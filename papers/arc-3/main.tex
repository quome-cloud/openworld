{\sffamily\bfseries\color{cardblue} ABSTRACT}\par
\smallskip
\fontsize{9.2}{10.5}\selectfont
\textbf{ARC-AGI-3} is the ARC Prize Foundation's interactive-reasoning benchmark: an agent dropped into
an unfamiliar grid game must discover the rules \emph{by acting} and reach goals over many steps;
frontier agents score near zero. We report a \textbf{source-free result that beats the prior SOTA on
every axis.} Under the benchmark's intended \textbf{source-free} protocol---process-isolated, no game
code; rules discovered by acting; per-run leakage audit; every solve replay-verified and round-tripped
through a verified OpenWorld \texttt{World}---a reasoning agent that writes, \emph{reasons the win
condition for}, and verifies its own world model completes \textbf{\ArcSFGames/\ArcUnionGames{} games}
(\ArcSFLevels/\ArcTotalLevels{} levels) at \textbf{\ArcRhaeSFGame{} per-game action-efficiency} (RHAE),
versus prior SOTA \emph{baseline1}'s source-free-intended \textbf{\ArcBaselineFull/\ArcUnionGames{}}
(145 levels, \ArcRhaeBaselineOne{} RHAE)~\cite{baseline1_ewm}---ahead on games, levels, and, most
decisively, \textbf{efficiency}: $\sim$54\% fewer actions per game, \ArcRhaeSFPerLevel{} per level, with
\ArcRhaeSFLevelsCapped/\ArcRhaeSFNLevels{} solved levels pinned at the human-efficiency cap (about a
\emph{third} of the human action count where we solve).

\textbf{Why it is hard---and the scientific contribution.} The bottleneck is \emph{not} modeling but
\textbf{goal discovery}: ARC-AGI-3 wins are \textbf{goal-as-procedure} (an ordered protocol), not
goal-as-state. \emph{Modeling is largely solved}---all \NumArcGames{} games are replay-deterministic
(\ArcDeterminismAll{}) and exactly code-expressible; a frontier synthesizer writes verified
\texttt{predict(frame,action)} code reaching \ArcClaudeFid{} mean held-out exact-match
(\ArcNNearPerfect{} near-perfect). Yet on top of those models, \emph{six} principled automated
goal-discovery methods (atomic objectives, frontier-LLM hypotheses, Bayesian sub-world planning, typed
navigation, a formal solving cycle, spectral exploration) solve \textbf{zero} hard games---several
\emph{even on a fidelity-$1.0$ model}---because a goal scored over a single \emph{state} cannot rank an
ordered \emph{procedure}. Only a reasoning agent crosses the walls, and only at \emph{fair budget}: the
same recipe trailed baseline1 until four \emph{harness/budget} confounds were removed (a frame-decode
crash, a per-game time cap, reasoning left at its default, and API rate-limit throttling); run fairly,
it overtakes prior SOTA.

Granting the agent \textbf{read access to each game's own source} (mechanics \emph{and} win condition)
lifts completion to \textbf{\ArcFullGames/\ArcUnionGames{}}; we present this not as the headline but as a
\textbf{source-access upper bound}---a security-relevant capability, since most critical infrastructure
is open-source. The \ArcSourceGapGames-game gap between protocols is the measured \emph{cost of
discovery}. Solves are offline with unbounded resets ($N{=}1$ per game); this is a \emph{fresh
fair-budget result and may be slightly revised}; random play alone clears $\ArcRandomGeqOne/\ArcUnionGames$
first levels, so we also report \textbf{\ArcAboveRandom/\ArcUnionGames{} above random}.
\textbf{Contributions:} (i)~the goal-as-procedure diagnosis, negatives as first-class results;
(ii)~a source-free result that \textbf{beats prior SOTA on games, levels, and---decisively---action
efficiency}; (iii)~an auditable source-free protocol whose telemetry makes leakage \emph{detectable},
every solver a serveable, verified OpenWorld \texttt{World}.

\smallskip
\noindent\textbf{Keywords:}~%
ARC-AGI-3; interactive reasoning; code world models; verification; planning.

\section{Introduction}
\textbf{The interactive frontier.} ARC-AGI-3~\cite{arcagi3_report} is the latest in the series. ARC-AGI-1 and -2 measure abstraction from static
input--output grids; ARC-AGI-3 moves to \emph{interactive} environments where the rules and goals
are revealed only through action. This rewards four capabilities the prior benchmarks do not
isolate---exploration, dynamics \emph{modeling}, goal-setting, and planning---and it is hard:
the benchmark is human-calibrated so that \textbf{humans solve $100\%$ of the environments while
frontier agents score below $1\%$} (March 2026)~\cite{arcagi3_report}. By design it ``avoids language and
external knowledge,'' testing Core-Knowledge priors and \emph{fluid adaptive efficiency on novel tasks}
``over memorized patterns''~\cite{arcagi3_report}---a stance that makes the \emph{source-free} protocol the
benchmark's intended condition (external knowledge, including the game's own code, is to be discovered, not
supplied) and that sharpens our training-memory caveat (\S\ref{sec:limitations}): a public game memorized in
frontier weights is exactly the ``memorized pattern'' the benchmark seeks to exclude. \textbf{Our central
finding inverts the natural thesis.} One would expect the bottleneck to be \emph{modeling}---an agent that
recovers the dynamics exactly can plan optimally. We show modeling is largely \emph{solved} (the dynamics
are exactly recoverable as verified code), and the true wall is \textbf{goal discovery}: ARC-AGI-3 wins are
\emph{goal-as-procedure} (an ordered protocol), which no score over a single state can rank, so principled
automated goal-discovery methods solve zero hard games even on a perfect model.

\textbf{This paper.} We port \openworld{}'s verified-code-world-model recipe to ARC-AGI-3 and use it
to \emph{diagnose} the benchmark. Our contributions, in order of importance: \textbf{(i)~the
goal-as-procedure diagnosis}---four principled goal-discovery methods (atomic objectives, frontier-LLM
hypotheses, Bayesian sub-worlds, typed navigation) solve \emph{zero} hard games, several even on a
fidelity-$1.0$ model, because a goal scored over a single \emph{state} cannot express an ordered
\emph{procedure}; \textbf{(ii)~the strategy landscape} (Fig.~\ref{fig:heatmap})---a guide to what does
and does not solve interactive reasoning, with the negatives as first-class results; \textbf{(iii)~an
honest above-random measure}---under the offline unbounded-reset protocol random play reaches
the first level on $\ArcRandomGeqOne/\ArcUnionGames$ (a \ArcRandomBudget{}-step/game budget; never a
full game), so we report \textbf{\ArcAboveRandom/\ArcUnionGames{} above random} rather than the inflated $\geq$1-level
union; and \textbf{(iv)~an auditable source-free protocol with a quantified discovery gap}---the same
executable-world-model recipe completes \textbf{\ArcFullGames/\ArcUnionGames{} games source-assisted}
and \textbf{\ArcSFGames/\ArcUnionGames{} source-free} (audited per-run), each solver a serveable
OpenWorld \texttt{World} (\S\ref{sec:fullgame}); the \ArcSourceGapGames-game gap is the measured cost
of discovery. On the source-free axis our \ArcSFGames{} now \emph{beats} baseline1's
\ArcBaselineFull{}~\cite{baseline1_ewm} (source-free-intended) on all three axes---games
(\ArcSFGames{} vs.\ \ArcBaselineFull{}), levels (\ArcSFLevels{} vs.\ 145), and, most decisively,
action-efficiency (\ArcRhaeSFGame{} vs.\ \ArcRhaeBaselineOne{} per-game RHAE). An earlier source-free
number trailed baseline1 only because of harness/budget confounds (a frame-decode crash, a
\ArcOurBudgetMin-min/game cap, reasoning left at default, and API rate-limit throttling); run
\emph{fairly} (fixed wrapper, full per-game budget, max reasoning, rate-limit-resilient), the \emph{same
recipe on a verified-World substrate} overtakes prior SOTA (\S\ref{sec:limitations}). The
source-assisted \ArcFullGames/\ArcUnionGames{} is a further upper bound on what code access buys.
(Limitations---offline, unbounded resets, $N{=}1$, and this being a fresh fair-budget result---are
consolidated in \S\ref{sec:limitations}.)

\paragraph{Three numbers, read carefully.} ARC-AGI-3 games have \emph{multiple} levels ($6$--$10$
each), so ``solved'' is ambiguous; we prefer \emph{full-game completion} over $\geq$1-level
reachability (which is nearly free---random play alone reaches one level on
$\ArcRandomGeqOne/\ArcUnionGames$). On completion we report \textbf{\ArcSFGames/\ArcUnionGames{}
source-free} (the benchmark's intended condition; beating baseline1's
\ArcBaselineFull/\ArcUnionGames{}~\cite{baseline1_ewm} on games, levels, and efficiency) and
\textbf{\ArcFullGames/\ArcUnionGames{}
source-assisted} as a source-access upper bound (\S\ref{sec:fullgame}); the \ArcSourceGapGames-game gap
is the measured cost of discovery. Limitations (offline, unbounded resets, $N{=}1$) are consolidated in
\S\ref{sec:limitations}.

\textbf{Prior art.} ARC-AGI-3~\cite{arcagi3_report} exposes a wide gap: frontier LLM agents score
under $1\%$, while purpose-built systems do far better (Table~\ref{tab:priorart}). The state of the
art, \emph{baseline1}~\cite{baseline1_ewm}, has a coding agent construct an \emph{executable} Python
world model with verifiers and an MDL simplicity bias---the recipe our live coding agent reproduces in
OpenWorld with Claude. \emph{Rigor and auditability caveats.} baseline1 is a sole-authored,
by-its-own-description ``intentionally direct'' systems report: a scripted controller over
\emph{predefined} world-model interfaces and verifier programs, with an unspecified ``MDL-like''
refactor, \emph{no} ablations (the verifier, refactor, planner, and base model are not separated), and
results on the \emph{public} set only---the private validation set, by the author's statement,
``remains to be tested.'' It targets a source-free protocol and reports
discarding results from an earlier, source-leaky harness version; but with the agent's ``broad
system access,'' its leakage ``audit'' is asserted rather than per-run quantified. We do not dispute the result---we build
on the recipe---but absent a per-run leakage audit, ablations, error bars, or reported cost, the figure
can be neither confirmed leak-free nor attributed to its components. That is precisely the gap our
protocol closes: per-run audited telemetry that \emph{quantifies} the leak channel (we catch our own
agent reading source), replay $+$ OpenWorld round-trip verification, two clearly-labeled protocols, and
reported cost.
\emph{DreamTeam}~\cite{dreamteam_workspace} casts the external workspace
(prompts, files, code) as the trainable substrate; the training-free \emph{Graph
Explorer}~\cite{graphexplore_arc3} maps each game as a directed state graph with status-bar masking and
salience-tiered clicks (its frame-processing directly informs our perceptors);
\emph{StochasticGoose}~\cite{stochasticgoose} predicts frame-change with a CNN. Harness-adaptation
work~\cite{harness_adapt} and the ARC Prize technical reports~\cite{arcprize2025_report} round out the
landscape. Our contribution is: an \textbf{auditable source-free protocol}---per-run leakage
audit, replay-verify, OpenWorld round-trip---that empirically shows a capable agent given source
takes the shortcut (caught by our own telemetry), yielding source-assisted
\ArcFullGames/\ArcUnionGames{} as an upper bound and audited source-free \ArcSFGames/\ArcUnionGames{}
(beating baseline1's \ArcBaselineFull{} on games, levels, and efficiency); \emph{plus} the \emph{diagnosis} (goal-as-procedure) and the
\emph{map} of which method class solves what. Crucially, every solver is an \textbf{open, serveable,
verified \texttt{World}}---the whole result downloads, re-runs, and serves from one zero-dependency
repo, which is the point: the recipe is reusable, not a one-off score.

\section{Two protocols: \ArcSFGames/\ArcUnionGames{} source-free, \ArcFullGames/\ArcUnionGames{} with game source}
\label{sec:fullgame}
We complete full games under \textbf{two protocols} that differ only in whether the agent may read the
game's own code (Fig.~\ref{fig:arc3source}); neither uses banked solution traces, and both are
replay-verified against the real engine. Per-protocol detail, the Codex head-to-head, the per-game
completion matrix, the OpenWorld atlas, and the security and efficiency analyses are in
App.~\ref{app:fullgame-detail}.

\textbf{Source-free (the benchmark-intended result).} The agent runs process-isolated with \emph{no}
access to the game source---it discovers the dynamics by acting and infers the win condition from frames.
A solve is banked only if it audits clean (no source read), replay-verifies, and round-trips through an
OpenWorld \texttt{World}. Under this harder protocol we complete \textbf{\ArcSFGames/\ArcUnionGames{}
games} (\ArcSFLevels/\ArcTotalLevels{} levels): \ArcSFList{}. This \emph{beats} baseline1's
source-free-intended \ArcBaselineFull/\ArcUnionGames{}~\cite{baseline1_ewm} (145 levels) on games and
levels, and---most decisively---on action-efficiency (\ArcRhaeSFGame{} vs.\ \ArcRhaeBaselineOne{}
per-game RHAE; \S\ref{sec:rhae}).

\textbf{Source-assisted (a source-access upper bound).} Granting the agent each game's downloaded source
(\texttt{<game>.py}: layout, dynamics, \emph{and} win condition---but no solution) to build a faithful
simulator and \emph{search} it lifts completion to \textbf{\ArcFullGames/\ArcUnionGames{}} games
(\ArcFullLevels/\ArcTotalLevels{} levels); \ArcPartialGames{} game remains partial (\ArcPartialList). We
present this \emph{not} as the headline but as an upper bound on what code access buys---and a
\textbf{security-relevant} one: the same recover-then-search capability applied to a real codebase (most
critical infrastructure is open-source) is a concrete cyber-offensive skill that scales with model
strength (App.~\ref{app:fullgame-detail}). The recipe is model-portable (Claude and Codex reach
\emph{identical} per-game levels on every game but \texttt{lf52}) and human-competitive on the
leaderboard's official action metric (mean \ArcRhaeMean{}; App.~\ref{app:fullgame-detail}).

\textbf{The gap is the result.} The \textbf{\ArcSourceGapGames-game, \ArcSourceGapLevels-level} delta
between the two protocols \emph{quantifies the cost of having to discover the world}: reading the code
hands the agent the exact dynamics and goal (so only search remains), whereas source-free must solve the
goal-as-procedure wall of \S\ref{sec:solvewall} from interaction alone. Reading a game's source reveals
its \emph{mechanics and win condition}, not the solution steps (the agent derives the action path either
way), so the gap isolates the cost of \emph{discovering} dynamics, not of planning them.
App.~\ref{app:pergame} (Table~\ref{tab:arc3games}) lists every game; limitations are consolidated in
\S\ref{sec:limitations}.

\section{The ARC-AGI-3 setting}
Each game presents a $64\times64$ grid of $16$ colors; the agent issues discrete actions and
receives the next frame, a game state (in-progress / win / over), and a level counter. We access
the \NumArcGames{} public games locally through the official \texttt{arc-agi} toolkit
(a Gym-like \texttt{reset}/\texttt{step} interface) and score with its scorecards. We treat each
game as a \emph{world}: a deterministic transition function over the symbolic grid state.
Fig.~\ref{fig:example} shows one game (\texttt{ka59}) and a verified solution end-to-end, to ground
what follows.

\section{Modeling is largely solved}
\label{sec:modeling}
The recipe is the \openworld{} loop. \textbf{Explore:} a budgeted policy (random, then change-seeking)
gathers $(s,a,s')$ transitions, each verified by construction (the environment is ground truth).
\textbf{Synthesize verified code:} an LLM writes a Python \texttt{predict(frame,action)}, accepted
\emph{only if} it exact-matches held-out transitions (a per-frame gate; it emits local update rules and
retries with counterexamples). \textbf{Plan:} the agent searches action sequences through the verified
code (BFS / CEM lookahead) ``in imagination'' at native speed. Run on all \NumArcGames{} games with three
synthesizers (qwen2.5-coder 7B/32B, Claude), this establishes that modeling is \emph{not} the wall.
\emph{The precondition is universal}: replay-determinism is \ArcDeterminismAll{} on every game, so the
dynamics are exactly writable as code. \emph{A frontier synthesizer recovers them}: Claude reaches
\ArcClaudeFid{} mean held-out exact-match on the \ArcNReal{} real-dynamics games (excluding trivial
no-change games), beating a copy-frame baseline on \ArcClaudeBeatsCopy/\ArcNReal{} with
\ArcNNearPerfect{} near-perfect ($\geq$90\%) world models, including dense games where copy is near-zero.
\emph{Capability is decisive}: the local model captures almost nothing (qwen-32B \ArcQwenThirtyTwoFid{},
qwen-7B \ArcQwenSevenFid{}), so the bottleneck is \emph{generation}, not determinism or verification
(detail, the fidelity figure, and honest limits in App.~\ref{app:modeling}). Fidelity is single-step and
is not solving ($0$ levels here, by design); solving is the rest of the paper.

\section{The solving wall: goal inference}
\label{sec:solvewall}

\begin{figure}[t]
\centering
\includegraphics[width=0.92\textwidth]{figs/arc3_goal_as_procedure.png}
\caption{\textbf{The diagnosis: goal-as-state vs.\ goal-as-procedure.} Left: methods that optimize a
score over a single \emph{state} (MPC, atomic objectives, LLM/Bayesian goal hypotheses) climb to the
wrong configuration. Right: the win is a specific ordered \emph{procedure} (A$\to$B$\to$C), which no

\label{fig:procedure}

\emph{Automated} solving (no reasoning agent in the loop) is \textbf{hard}, and the value of this
study is locating exactly why---a diagnosis that later tells the router (\S\ref{sec:router}) which
games to escalate to a live coding agent (\S\ref{sec:agent}), which crosses this wall on every one. (1)~\emph{Exploration cannot find a reward.} With a perfect model, neither pixel-novelty
nor object-graph-novelty exploration completes a level: both saturate a tiny reachable region
(\ArcExploreLo--\ArcExploreHi{} states) and trigger \emph{zero} reward---the win is not reachable by
undirected action, regardless of representation. (2)~\emph{Goal hypotheses are sophisticated but
unverifiable.} Shown the dynamics and sample frames, Claude infers coherent goals (for \texttt{sp80}
it identifies the movable block, the target receptacles, and explicitly ignores the timer), yet
one-shot they are guesses---on \texttt{s5i5} it mistakes a countdown \emph{timer} for the objective
and optimizes toward losing. (3)~\emph{Closing the loop is not enough.} Feeding real-env feedback
(``optimizing that goal ended the game with $0$ wins'') still fails to solve (\texttt{s5i5}
$0/8$; the only level reached, \texttt{sp80}'s first, is trivially reachable by random too): negative
feedback says what is \emph{wrong}, not what is \emph{right}, and without ever observing one positive
reward there is nothing to steer toward. The bottleneck is thus neither modeling, planning, nor
exploration coverage but \textbf{goal inference}---discovering the win condition with no reward
signal. This is the wall frontier agents also hit ($\sim$0 on ARC-AGI-3), here precisely located:
for \emph{automated} verified-world-model agents this is the central wall---and
\S\ref{sec:agent} shows that a live coding agent which \emph{reasons} the win condition (rather than
searching a fixed hypothesis space) crosses it on every such game.

\section{The strategy landscape: what solves ARC-AGI-3}
\label{sec:landscape}
We ran a dozen increasingly sophisticated strategies and record, per game, which produce a verified
($\geq$1-level, replay-checked) solve. The landscape (Fig.~\ref{fig:heatmap}) is the paper's most useful
artifact for world-model research---it shows where each idea works and, just as importantly, where
principled ideas fail. Three \emph{categories} of approach emerge.

\textbf{Goal-discovery methods---all fail.} Multiple principled, automated attacks that try to
\emph{infer the win condition} with no reasoning agent each solve \textbf{0} walled games, several
\emph{even on a fidelity-$1.0$ model}: atomic candidate-goal objectives; frontier-LLM goal hypotheses
(open- and closed-loop); procedure-aware Bayesian active inference over composite sub-worlds; a
source-free engine-\emph{reconstruction} pipeline with a held-out Clopper--Pearson certificate; an
explicit convergence-guaranteed formal solving cycle; and graph/spectral (Fiedler-bottleneck)
exploration (a short-horizon receding-horizon variant is in progress). Together they rule out a large
class of approaches and sharpen the diagnosis: a goal scored over a single \emph{state} cannot rank an
ordered \emph{procedure}, so undirected search occasionally stumbles into a win while goal-conditioned
planning never does---and a \emph{forming} world model (many distinguished states) is worthless until it
contains a \emph{verified} win-terminating path (Fig.~\ref{fig:forming}). Per-experiment detail---and the
methodological dividend that the engine-reconstruction certificate yields a shared-prior-bias
diagnostic---is in App.~\ref{app:negatives}.

\textbf{Perception and search---partial coverage.} Directed search and graph exploration each reach $6$
games; multi-perception consensus reaches $10$ (uniquely \texttt{cn04, lf52, m0r0, s5i5}). These solve
the \emph{reachable} games but not the procedural walls (App.~\ref{app:consensus}).

\textbf{The reasoning agent---crosses the walls} (\S\ref{sec:agent}), the only method that does. A
reachability \emph{router} (App.~\ref{sec:router}) sends each game to the cheapest solver that clears
it---cheap perceptual/graph solvers when they suffice, the agent for the walls---combining the
strategies into one pipeline that is $\geq$1 level on all \ArcUnionGames{} (\ArcFullGames/\ArcUnionGames{}
full source-assisted, \ArcSFGames/\ArcUnionGames{} source-free).

\section{Cracking the walls: a reasoning agent that writes its own world model}
\label{sec:agent}
The procedural walls resist every automated attack above because they need \emph{genuine goal
reasoning}, not search over a fixed hypothesis space. Handing each game to a \textbf{live coding agent}
crosses them: Claude Code, given the deterministic harness and the OpenWorld recipe---explore
$(s,a,s')$, write a verified \texttt{predict(frame,action)} model, \emph{reason} what raises
\texttt{levels\_completed}, plan, and replay-verify (a synthesized \texttt{CodeTransition} plus a
reasoned \texttt{CodeObjective}). The agent solves \textbf{15/25} games on its own---every wall the
automated methods missed (\texttt{dc22, ft09, g50t, ka59, lp85, re86, sb26, sc25, su15, tn36,
wa30})---lifting the union across all strategies to $\geq$1 level on every public game; chaining levels
then yields the full-game completions of \S\ref{sec:fullgame}. (The agent solves 15 standalone; the
deployed router, App.~\ref{sec:router}, invokes it only on the \textbf{13} games no cheaper solver
clears---the rest go to cheap perceptual/graph solvers.) We do not claim every solve reflects deep
reasoning: some are clearly deliberate (\texttt{ka59} navigates then clicks a door; \texttt{dc22}
interleaves moves and clicks), others determinism-exploiting \emph{sweeps} the offline protocol permits
(\texttt{lp85} repeats one click; \texttt{ft09} tiles a grid); the random-play baseline
(\S\ref{sec:baseline}) separates the two. The model-comparison (Claude vs.\ Codex), the SOTA
comparison, and the cost/efficiency discussion are in App.~\ref{app:fullgame-detail}.

Because every solve here is \emph{offline} with \emph{unbounded resets}, we must ask how much the
\emph{protocol}---not the method---contributes. We run uniform random play (random directional actions
or random $(x,y)$ clicks, resetting on episode end) under a fixed budget of \textbf{\ArcRandomBudget{}
steps per game} (each episode capped at \ArcRandomMaxDepth{} steps before a reset). \emph{This baseline
measures reaching the first level only, never full-game completion.} \textbf{Random clears at least one
level (its $\geq$1-level reach) on \ArcRandomGeqOne/\ArcUnionGames{} games} (\texttt{ar25, cd82, cn04,

levels vs.\ 2; \texttt{r11l} 6 vs.\ 1). We thus report two numbers and prefer the second:
$25/25$ at $\geq$1 level (\ArcRandomGeqOne{} random-reachable) and \textbf{\ArcAboveRandom/\ArcUnionGames{} above random}.

\section{Limitations}
\label{sec:limitations}
We state, once and plainly, the boundaries of every result above.
\begin{enumerate}\setlength{\itemsep}{2pt}
\item \textbf{Offline, unbounded resets---not the live leaderboard.} All solves are computed offline
against the deterministic environment with \emph{unbounded} resets and deterministic replay-verification;
we report games \emph{completed}, not the leaderboard's live action-\emph{efficiency} penalty (RHAE). Our
action economy is our weak axis: long replay sequences and free resets mean an explicit live RHAE score
would be lower than the offline efficiency figure of App.~\ref{app:fullgame-detail}. The confirming live
run is a labeled next step.
\item \textbf{Source-assisted reads the win condition.} The \ArcFullGames/\ArcUnionGames{} source-assisted
number is obtained with the agent reading each game's own code---its \emph{mechanics and win condition}
(though not the solution steps). It is a source-access \emph{upper bound}, not directly comparable to the
benchmark-intended source-free protocol; the human-comparable result is the source-free
\ArcSFGames/\ArcUnionGames{}.
\item \textbf{Single run; no variance reported ($N{=}1$ per game).} Each game's reported result is a
\emph{single} solve, not an average over seeds or agent sessions; we report no error bars or success-rate
distribution. Per-game outcomes may vary across runs of the stochastic agent, and the headline counts
should be read as one realized run, not an expectation.
\item \textbf{Training-memory contamination is unmeasured.} The protocol audits that the agent never
\emph{reads} game source \emph{at run time}; it cannot rule out that a game---public on GitHub and the
leaderboard---sits in the \emph{model's training data}. This channel is unverifiable (one cannot inspect
the weights) and affects \emph{every} frontier-model result on this benchmark, prior SOTA and the live
leaderboard included. The strongest claim the protocol supports is therefore \emph{no runtime source
access (audited)}, not ``no prior knowledge''; a contamination-free test needs a model whose cutoff
provably predates a game's release (the direction of our local-model controls).
\item \textbf{Fresh fair-budget result (how the source-free comparison became a win).} An earlier
source-free run trailed baseline1 only because of harness/budget confounds---a frame-decode crash, a
\ArcOurBudgetMin-minute wall-clock cap per game per round over \ArcOurBudgetRounds{} rounds, the Codex
arm's reasoning effort left at its \emph{default} (never raised), and API rate-limit throttling.
baseline1~\cite{baseline1_ewm} reports \emph{no} budget cap. Re-run \emph{fairly}---fixed wrapper (crash
removed), full per-game budget, maximum reasoning effort, and rate-limit-resilient retries---the same
recipe on a verified-World substrate completes \ArcSFGames/\ArcUnionGames{} games (\ArcSFLevels{} levels)
at \ArcRhaeSFGame{} per-game RHAE, overtaking baseline1's \ArcBaselineFull/\ArcUnionGames{} (145 levels,
\ArcRhaeBaselineOne{}) on all three axes---games, levels, and (widest) efficiency. We report this as a
\emph{fresh} result (the fair-budget sweep is still finishing, so numbers may be slightly revised) and it
remains offline, unbounded-reset, and $N{=}1$ per game.
\item \textbf{Cost is an uncontrolled estimate.} The whole campaign (all \ArcUnionGames{} games) ran from
a single Claude Code subscription over $<\!24$h; prorating the \$200/month plan gives $\approx$\$7.
baseline1 reports running on a comparable \$200/month subscription that covered ``roughly two to eight
games''---so at the same subscription price our run covers the full suite---but we did not meter tokens
and the engines and accounting differ, so we avoid a per-run dollar multiple and treat this as an
estimate, not a controlled figure.
\end{enumerate}

\section{Reproducibility}
\label{sec:reproducibility}
The agent, the per-game transition logs, and every number regenerate from one repository; the
ARC-AGI-3 environments are the official public games accessed via the \texttt{arc-agi} toolkit. Each
experiment is a self-contained script under \texttt{experiments/}: synthesis (\texttt{e86\_arc3.py},
\texttt{e86b\_agentic.py}), planning/controllability (\texttt{e87\_arc3\_plan.py}), the solving
pipeline---reward capture (\texttt{e93\_capture\_reward.py}), deterministic replay-verify
(\texttt{e93b\_replay\_solve.py}), directed-vs-random (\texttt{e93d\_directed\_vs\_random.py}), and the
\ArcNSolved-game interact-biased sweep (\texttt{e99\_solve\_sweep.py})---the world-model-driven solver
(\texttt{e101\_model\_solver.py}), and the goal-discovery attack (\texttt{e102\_goal\_search.py}). The
foundational primitives are in the zero-dependency core: \texttt{CodeObjective}/\texttt{induce\_reward}
(\texttt{openworld/reward.py}), \texttt{ConsensusTransition} (\texttt{consensus.py}), and
\texttt{make\_typed\_perceptor} (\texttt{perceive.py}). \textbf{The verified winning action sequences
for all \ArcNSolved{} solves are saved in \texttt{experiments/results/e99\_deep\_sweep.json}} and
replay-verify deterministically against the live games; all numbers and figures regenerate via
\texttt{scripts/make\_arc3\_assets.py}. A one-command check,
\texttt{bench/verify\_arc3\_solves.sh}, replays every saved winning sequence against the live games
and confirms all \ArcNSolved{} complete a level.

\paragraph{Full-game result, efficiency, and OpenWorld round-trip.} The full-game numbers regenerate
the same way. \textbf{Source of truth:} \texttt{scripts/build\_arc3\_fullgame.py} reads each game's
replay-verified solution and writes \texttt{experiments/results/arc3\_fullgame.json}
(\ArcFullGames/\ArcUnionGames{} full, \ArcFullLevels/\ArcTotalLevels{} levels); \texttt{make\_arc3\_assets.py}
turns it into the completion matrix (Fig.~\ref{fig:arc3matrix}), the per-game table
(Table~\ref{tab:arc3games}), the atlas gallery (Fig.~\ref{fig:arc3gallery}), and every \texttt{Arc*}
macro. \textbf{Action efficiency (E120):} \texttt{experiments/e120\_rhae.py} replays each solution in
the engine, segments actions per level, and scores them with \texttt{arc\_agi}'s \emph{own}
\texttt{EnvironmentScoreCalculator} against the official per-level human baselines shipped in
\texttt{EnvironmentInfo.baseline\_actions}---mean \ArcRhaeMean{} (Fig.~\ref{fig:arc3rhae},
\texttt{arc3\_rhae.json}); no metric is re-implemented. \textbf{OpenWorld round-trip (E121):}
\texttt{experiments/e121\_openworld\_roundtrip.py} builds, per game, an \texttt{openworld.World} from the
discovered masked-frame state graph (a \texttt{FunctionTransition} over the learned table plus an
induced \texttt{CodeObjective}, reward $=$ \texttt{levels\_completed}) and \emph{replays the banked
solution through it} via \texttt{world.step}; all \textbf{\ArcOWVerified/\ArcOWTotal{}} reproduce their
verified level count through the framework, and every world \texttt{to\_spec}/\texttt{validate\_spec}/
\texttt{render\_card} cleanly (\ArcOWIndexed{} game uses trajectory-indexed state where status-bar
masking is too coarse; flagged in \texttt{arc3\_openworld\_roundtrip.json}). The two solver scripts run
under the \texttt{arc-agi} interpreter; the world round-trip imports the zero-dependency
\texttt{openworld} core. In short:
\texttt{build\_arc3\_fullgame.py $\to$ e120\_rhae.py $\to$ e121\_openworld\_roundtrip.py $\to$
make\_arc3\_assets.py} regenerates every full-game number, figure, and verification in this paper.

\section{Two protocols: full detail, parity, security, and efficiency}
\label{app:fullgame-detail}
This appendix expands the condensed two-protocol result of \S\ref{sec:fullgame}: the source-faithful
construction, the Claude-vs-Codex head-to-head, the security reading, the per-game completion matrix and
OpenWorld atlas, and the official action-efficiency analysis.

\textbf{Source-faithful construction.} The agent reads each game's downloaded source (\texttt{<game>.py}:
map layout, dynamics, and win condition---but \emph{no} solution) to build a faithful simulator,
\emph{searches} for the winning procedure, and \emph{replay-verifies} it. We fully complete
\ArcFullGames{} of the \ArcUnionGames{} public games (\ArcFullPct), \ArcFullLevels/\ArcTotalLevels{}
levels (\ArcLevelsPct). Because the agent reads each game's source, this source-\emph{assisted} number is
\emph{not} the like-for-like comparison to baseline1's source-free-intended
\ArcBaselineFull/\ArcUnionGames{}~\cite{baseline1_ewm}---that comparison is our source-free
\ArcSFGames/\ArcUnionGames{}, which beats baseline1 on games, levels, and efficiency
(\S\ref{sec:rhae}). The source-free protocol (\S\ref{sec:fullgame})
banks a solve only if it audits clean (no source read), replay-verifies, and round-trips through an
OpenWorld \texttt{World}.

\textbf{Model-portable: Claude and Codex agree.} The source-assisted recipe is not specific to one model.
Running the same protocol with \ArcCodexModel{} (Codex) completes \ArcCodexFullGames/\ArcCodexUnionGames{}
games---matching Claude---and the per-game level counts are \emph{identical on every game except}
\texttt{lf52} (Fig.~\ref{fig:arc3codex}). On \texttt{lf52} Codex reaches level~\ArcCodexLfTwoLevels{}
versus Claude's level~\ArcClaudeLfTwoLevels{} (neither completes its \ArcCodexLfTwoWin{} levels), so Codex
totals \ArcCodexLevels/\ArcCodexTotalLevels{} levels against Claude's \ArcFullLevels/\ArcTotalLevels{}---a
single-game, three-level edge and the only difference between the two runs. Codex is also notably
command-efficient (\ArcCodexCmdLo--\ArcCodexCmdHi{} agent commands per game). The takeaway is near-total
parity: both capable agents recover each game's dynamics from source and search to completion, with the
lone open game (\texttt{lf52}) hard for both.

\textbf{Why the distinction matters: humans cannot read source.} ARC-AGI-3 is a \emph{human-vs-machine}
benchmark---human players solve these games at near-ceiling by \emph{playing}, never by inspecting code,
while public agents stall~\cite{arcagi3_report}. The source-free protocol is therefore the one directly
comparable to that human baseline (both discover the rules by acting); the source-faithful result instead
measures the advantage of an information channel \emph{no human player has}. Our two numbers bracket the
axis: \ArcSFGames/\ArcUnionGames{} is the human-comparable result, and \ArcFullGames/\ArcUnionGames{} is
what becomes possible the moment a system's code is exposed to a capable agent.

\textbf{A security reading.} Each ARC-AGI-3 game is a small adversarial system with hidden dynamics and a
hidden objective. The source-faithful result shows that an agent combining (i)~a capable reasoner with
(ii)~read access to a system's source can recover its exact behavior, build a faithful simulator, and
\emph{search} it to any objective---here, completing every level. For real software the implication is a
caution: when \emph{both} model intelligence and target source are available to an autonomous agent, the
methods here let it exhaustively solve---or exploit---the system, a capability that scales with model
strength. \textbf{This is not a contrived protocol: it mirrors a real and growing threat surface.} The
world's critical infrastructure runs largely on \emph{open-source} software (Linux, OpenSSL, countless
libraries), where the source \emph{is} public---so an agent that performs source-assisted goal discovery
over a real codebase (recover behavior, then \emph{search} it to an objective such as a privilege
escalation) is a concrete cyber-offensive capability, not an artifact of our setup. The source-assisted
result therefore measures a genuinely consequential agentic skill for the present era, even as the
defensible, human-relevant evaluation condition is source-free---which is exactly the harder protocol
where performance drops by \ArcSourceGapGames{} games.

\textbf{Audited leak channel: no network, no web tools.} The source-free protocol forbids the agent from
\emph{fetching} anything---the rules must be discovered by acting, so a single network call (to a
walkthrough, the game's source, or a solution) would void the result. We audit that channel directly
rather than asserting it. A read-only pass over every captured source-free transcript
(\ArcAuditTranscripts{} runs across the opus and fable arms) scans two things: every tool invocation, and
every one of the \ArcAuditBashCmds{} shell commands the agents issued. The tool inventory is entirely
local---\texttt{Bash}, \texttt{Read}/\texttt{Write}/\texttt{Edit}, task and skill calls---with
\ArcAuditWebCalls{} invocations of any \texttt{WebSearch}/\texttt{WebFetch}/browser tool; and across all
\ArcAuditBashCmds{} commands there are \ArcAuditNetHits{} unambiguous network operations (no
\texttt{curl}/\texttt{wget}, URL, HTTP client, socket connect, package install, or \texttt{git} remote
op). The agents solve entirely offline: reading frames from the process-isolated sandbox, writing and
running local Python, and reasoning---consistent with the harness design (the agent's interpreter cannot
even \texttt{import arc\_agi}). This is the leakage channel made \emph{measurable}: the same telemetry
would name any offending run, and here it names none (\S\ref{sec:reproducibility}; auditor
\texttt{scripts/audit\_network\_tools.py}).

\textbf{Every solve is a verified, serveable artifact.} Each game's solver is an \openworld{}
\texttt{World}: a masked-frame perceptor maps pixels to symbolic state, a \texttt{FunctionTransition} over
the learned dynamics is the model, and \texttt{levels\_completed} (or an induced \texttt{CodeObjective})
is the reward. We verify this end-to-end: for every game we build the \texttt{World} from the discovered
masked-frame state graph and \emph{replay the banked solution through it} (\texttt{world.step} per

OpenWorld world}, and every world serializes to a valid spec and renders the atlas card in
Fig.~\ref{fig:arc3gallery} (one game needs trajectory-indexed state where status-bar masking is too
coarse). The solves thus \emph{run inside the framework}, and each downloads, re-runs, and serves from one
zero-dependency repo under \texttt{openworld serve}.

\subsection{Action efficiency: human-competitive (mean \ArcRhaeMean{})}
\label{sec:rhae}
A full-game count says we \emph{solve}; the leaderboard also asks how \emph{efficiently}. ARC-AGI-3
scores each level $\min\!\big((b/a)^2\cdot 100,\ 115\big)$, where $b$ is the \emph{human} baseline
action count (shipped per level in the environment) and $a$ is the agent's actions, index-weighted into
a per-game score---$100$ means human-efficient. \textbf{Scoring our verified solutions with the
engine's own calculator and the official human baselines, we average \ArcRhaeMean{}} across
\ArcRhaeN{} games---\textbf{\ArcRhaeAboveHuman{} of \ArcRhaeN{} pinned at the human-efficiency cap} and
\textbf{\ArcRhaeAboveBaseline{} at or above baseline1's reported \ArcRhaeBaselineOne}---with per-level
action counts typically a \emph{third} of the human baseline (Fig.~\ref{fig:arc3rhae}). This is not an
accident of the approach but its point: \emph{an exact, verified world model is precisely what lets you
search for short plans}, where an approximate model cannot. These figures pertain to the
\textbf{source-assisted} solves.

\textbf{Source-free efficiency---the decisive axis over baseline1.} We also score the
\emph{audited source-free} solves (replayed and split per level against the same official human
baselines), and this is the head-to-head that matters. On the levels we complete source-free, efficiency
is essentially human-capped: a mean per-level score of \textbf{\ArcRhaeSFPerLevel{}} with
\textbf{\ArcRhaeSFLevelsCapped/\ArcRhaeSFNLevels{} levels at the human cap}---the source-free solves are
\emph{as action-efficient as the source-assisted ones}. The official-style per-game source-free RHAE is
\textbf{\ArcRhaeSFGame{}}. Against prior SOTA this is a decisive win: baseline1 reports
\textbf{\ArcRhaeBaselineOne{}\%} per-game RHAE (source-free-intended)~\cite{baseline1_ewm}, so our
source-free solving is \emph{$\sim$54\% more action-efficient} (\ArcRhaeSFGame{} vs.\
\ArcRhaeBaselineOne{})---and, unlike the earlier budget-constrained run, we now also complete \emph{more}
games (\ArcSFGames/\ArcUnionGames{} vs.\ \ArcBaselineFull/\ArcUnionGames{}) and \emph{more} levels
(\ArcSFLevels{} vs.\ 145). Efficiency is the widest margin, and where we solve, we solve tightly.

\textbf{Honest caveat.} This scores the \emph{executed} solution, not the offline search that found it.
If the protocol permits offline precomputation---the downloadable environment provides each game's
source---then executing these plans live realizes the score directly; if it requires online discovery,
in-run exploration also counts and \ArcRhaeMean{} is an upper bound. Either way it is the \emph{same}
formula and the \emph{same} official baselines the leaderboard uses; the confirming run---executing our
precomputed plans through the live API and reading the scorecard---is the labeled next step
(\S\ref{sec:limitations}).

\medskip
\noindent\textbf{RHAE is a solution-\emph{path} economy metric, not a cost-to-\emph{solve} metric.}
To be fully explicit about what the number does and does not include: RHAE scores \emph{only the banked
solution's replay path}---the action count $a$ of the final plan measured against the human baseline. It
counts \emph{none} of the cost of \emph{discovering} that plan: the exploration env-steps spent probing
dynamics, the LLM sampling and tokens the agent consumed while reasoning, and the build-plus-search of
the code world model itself. Figure~\ref{fig:rhaescope} draws this boundary. That discovery cost is
substantial and entirely orthogonal to the replay economy: on the source-free arm, \emph{discovering}
each solved game cost \$\ArcDiscCostOpus{} (opus) and \$\ArcDiscCostFable{} (fable) on average
(\ArcDiscTokOpus{}M vs.\ \ArcDiscTokFable{}M LLM tokens across the arms; Fig.~\ref{fig:disccost})---none
of which RHAE counts. So RHAE should be read as ``given a solution, how tight is its executed path,''
not ``how cheap is it to reach a solution.'' The head-to-head against baseline1 remains
apples-to-apples: both pipelines are scored on the identical official metric over the same human
baselines, and neither side's RHAE folds in discovery cost, so the $\sim$54\% path-efficiency margin is
a like-for-like comparison of the \emph{same} economy axis.

\medskip
\noindent A deeper caveat follows from \emph{how} a code-world-model agent perceives. Its native
modality is the code it writes: it does not observe the board the way a human sees a screen, but
\emph{through} the parser and simulator it synthesizes, and it ``acts'' by searching that simulator
offline. The replayed action sequence is therefore the compiled \emph{readout} of a search, not a
stream of embodied decisions commensurable with a human's---and it is produced at a code-writing
throughput no human approaches. Read as a comparison of \emph{process} (how efficiently a solver
\emph{thinks}), a human-action-relative score is a category error for such an agent: the units are
incommensurable by construction. RHAE survives only as an \emph{artifact} comparator---a
method-agnostic measure of how tight the submitted \emph{path} is, indifferent to the process that
produced it (which is exactly why a leaderboard can score humans, RL nets, and code-world-model agents
on it uniformly). The efficiency axis that \emph{is} commensurable for these agents is the discovery
cost itself (Fig.~\ref{fig:disccost}), measured in their own currency---LLM tokens, simulator rollouts,
dollars---rather than in human-action equivalents.

\section{Characterizing solutions: run lineages (E150)}
\label{app:lineage}

captured transcripts, so we characterize the process by what the agents \emph{do}: the code they write (a
median of \ArcLineageCodeKB{}\,KB per run), the tool/timestamp trajectory, and the banked action sequences.
Mining \ArcLineageRuns{} captured source-free runs (opus $+$ fable) yields nine views

\section{Goal-discovery negatives in detail}
\label{app:negatives}
The category-level summary of \S\ref{sec:landscape} is expanded here per experiment. All are
\emph{search} over a fixed hypothesis space, and all solve $0$ walled games.

\textbf{The goal-discovery trilogy (E102--E104).} (i)~a core-knowledge perceptor generating atomic
candidate-goal \texttt{CodeObjective}s, (ii)~a frontier model (Claude) generating rich, compositional,
closed-loop-refined goal hypotheses, and (iii)~procedure-aware Bayesian active inference over composite
sub-worlds with semiring-weighted planning---each solves \textbf{0} of the walled games, \emph{even with
a perfect (fidelity-1.0) model}, sharpening the diagnosis that a single-state goal score cannot express an
ordered procedure.

\textbf{Source-\emph{simulated} reconstruction (E127).} Rather than read a game's code, can two models
\emph{reconstruct} its engine from source-free play---synthesizing a stateful \texttt{Engine}, converging
them by differential CEGIS with the \emph{real environment} as the oracle, and \emph{certifying} the
result by a Clopper--Pearson lower bound on held-out next-frame accuracy? On the easy game \texttt{ar25}
this works cleanly: the certified engine reproduces \ArcSimEasyExact{} held-out transitions exactly
($\geq\ArcSimEasyBound$ at 95\% confidence). But on the gap game \texttt{dc22}---a wall the source-free
\emph{observational} solver already misses---the reconstruction collapses to \ArcSimGapExact{}
($\ArcSimGapAcc$) and the certificate \emph{correctly refuses to pass}: the toggle/counter dynamics are
not recoverable from shallow source-free play in a few rounds. Of \ArcSimAttempted{} games attempted,
\ArcSimCertified{} certified, and the method produces \emph{no} verified solves: reconstruction is easy
where the game is easy and fails exactly on the walls that motivate it. One methodological dividend
survives. Because the real environment---not two-model agreement---is the convergence oracle, the
\emph{A-vs-B-vs-real gap} ($+\ArcSimBias$ on \texttt{dc22} vs.\ $0$ on \texttt{ar25}) measures the
shared-prior bias by which two frontier models agree with \emph{each other} more than with reality,
precisely where the reconstruction is unreliable---a failure mode a naive two-model consensus would hide,
and a reminder that ground truth, not model agreement, must gate any source-free claim.

\textbf{An explicit formal solving cycle (E130).} Does giving the search a principled,
convergence-guaranteed structure help? E130 implements the behavioral cycle of a world-model formalism as
a source-free solver---an introspective world model $\sigma_I$, perception $\sigma_E$, an expert-pooled
\emph{moral filter} that selects actions, and a tension signal $T=\lVert\sigma_I-\sigma_E\rVert$ that
updates the model by acting. Its two load-bearing theorems \emph{reproduce} deterministically:
expert-facilitated selection reaches target accuracy in \ArcShuExpertConsult{} consultations versus an
amateur (random) filter's \ArcShuAmateurTrials{}---a \ArcShuSeparation$\times$ separation, with
$O(1/\sqrt N)$ error ($\ArcShuErrReduction\times$ reduction at $N{=}100$) against the amateur's
$\Theta(M)$---and the cycle contracts geometrically ($\rho=\ArcShuRho$) to a faithful model. This is the
formal statement of ``directed beats undirected.'' Yet \emph{online it cracks nothing}: across
\ArcShuOnlineGames{} the solver---even with the reinforcement-learning upgrades it invited (avatar
navigation, an adaptive expert bandit, object-relative dynamics; \ArcShuTests{} unit tests)---improves
\ArcShuOnlineGain{} of \ArcShuOnlineN{}. The mechanism demonstrably \emph{engages} (on \texttt{ka59} the
receding-horizon navigation executes \ArcShuNavSteps{} real steps over \ArcShuNavCycles{} cycles, not the
greedy single steps of the click-only baseline), but the heuristic proposers never \emph{propose} the
winning procedure: the EFEI theorem reduces \emph{variance}, not the \emph{bias} of a misspecified goal.

\textbf{Graph/spectral exploration (E135).} Is the lever a \emph{smarter frontier}---steering exploration
toward graph bottlenecks rather than expanding blindly? On the wall \texttt{\ArcSpectralGame} we compare a
blind forward walk against graph-novelty prioritization and against Fiedler (graph-Laplacian) bottleneck
ranking, which orders the frontier by the second-smallest eigenvector of the discovered state graph to
push through structural chokepoints. Under a matched \ArcSpectralBudget-step budget all three saturate the
same tiny region (\ArcSpectralBlindStates/\ArcSpectralNoveltyStates/\ArcSpectralFiedlerStates{} states)
and produce \emph{\ArcSpectralLevelGains{} level-gains}---the spectral and novelty heuristics even
discover slightly \emph{fewer} states than the blind walk, and none crosses a level. ARC-AGI-3 is
engineered against blind exploration on two axes at once: later levels explode the state space so coverage
is hopeless, and the official score is an action-\emph{efficiency} measure, so a graph-mapper that burns
hundreds of wasteful actions sees its score collapse even when it does complete a level
(Fig.~\ref{fig:forming}).

\textbf{A direction in progress (E131): short-horizon receding-horizon lookahead.} The negatives above all
try to infer a \emph{whole-level} goal from sparse feedback. E131 inverts the bottleneck: the explosion is
in the \emph{horizon}, not the branching (per step there are only $\sim$4--7 actions), so rather than
decode a global objective it exhaustively expands a \emph{2--3-frame} tree, scores leaves by
\emph{level-delta} (primary) then \emph{novelty} (tie-break), commits only the first action, and recedes
the window. A deterministic fast-forward transition cache makes the tiny tree nearly free---the
architecture is \emph{Lazy Receding-Horizon A\*}, with environment replay as a lazily-evaluated edge.
Whether chaining locally-optimal short lookaheads clears procedural walls that defeat whole-level goal
inference is the open question we are currently evaluating.

\section{Verified-code fidelity in detail (E86)}
\label{app:modeling}
Fidelity is \emph{held-out transition exact-match}: the fraction of unseen $(\text{frame},\text{action})$
pairs whose \emph{entire} next $64\times64$ frame the synthesized \texttt{predict} reproduces exactly. We
report on the \ArcNReal{} \emph{real-dynamics} games---excluding \ArcNTrivial{} games where random actions
trigger no state change (so ``predict no-change'' trivially scores $1.0$) and \ArcNExcluded{} game
(\texttt{tn36}) where random play never produced a valid step. Claude's mean lift over copy-frame is
\ArcClaudeLift{}, and Claude beats qwen-32B on \ArcClaudeBeatsQwen/\ArcRealForQwen{} games
(Fig.~\ref{fig:arc3fid}).

\section{Agentic synthesis and the capability ceiling (E86b)}
Does a better \emph{harness} close the synthesizer gap? We replace one-shot prompting with a

loop---the bottleneck is generation capability, not the harness.

\section{Planning through the model (E87)}
A verified model is only useful if you can \emph{search} it. We test \textbf{controllability}: sample
a reachable target (a short random action sequence from reset), plan a sequence \emph{through the
verified code} to reach it, then execute in the real environment. On a perfect model (\texttt{sb26},
fidelity $1.0$) planning reaches the target \ArcCtrlPlan{} of the time vs.\ \ArcCtrlRand{} for
random---the model is plannable. But success is \textbf{fidelity-gated}: below fidelity $1$ the plan,
correct \emph{in the model}, diverges in reality as single-step error compounds over the horizon, so
partial models do not yet support reliable multi-step control. \emph{Exact models admit search;
approximate ones mislead it}---reinforcing why verification, not just learning, is the point.

\section{Solving a level (E93): directed beats random}
Closing the loop from modeling to a genuine completion. A reachability sweep finds \texttt{\ArcSolveGame{}}
is the only public game whose reward is reachable by undirected play; reward-capture records a winning
episode, and deterministic replay verifies an \ArcSolveLen-action sequence that completes
\textbf{level \ArcSolveLevel{} of \texttt{\ArcSolveGame{}}}, reproducibly. This is a \emph{directed}
solve that beats random: at a matched \ArcSolveLen-step budget the verified solution succeeds
\textbf{\ArcDirectedRate{}} vs.\ \textbf{\ArcRandomRate{}} for random (\ArcSolveTrials{} trials).
\emph{Honest limits:} level 2 is unreachable even by a 600k-step biased search, and the solve is a
verified action \emph{sequence}, not a reverse-engineered win \emph{rule} -- the goal-inference wall
(\S\ref{sec:solvewall}) holds for the \emph{automated} methods here, though a reasoning agent
(\S\ref{sec:agent}) later completes full games (e.g.\ \texttt{ft09}, \texttt{r11l}).

\paragraph{Scaling to more games (E99): \ArcNSolved{} solved.} The sp80 win fired on an
\emph{interact} action, which a uniform sweep under-weights. An interact-biased, multi-seed reward
search (capture $\to$ deterministic replay-verify) per game raises the count from $1$ to
\textbf{\ArcNSolved{}/25 verified level-1 solves}: \ArcSolvedList{} (winning sequences
\ArcSeqLenRange{} actions, all saved and replay-verified). A solve counts only if its captured
sequence reproduces the completion deterministically. One of these (\texttt{cd82}) is additionally
solved \emph{through the synthesized code world model} (E101: predict-lookahead MPC to navigate the
typed agent), and the verified-reward induction (E97/E98) recognizes the win on \texttt{sp80} with
acc $1.0$ from the agent's \emph{own} synthesized objective.

\paragraph{What the remaining $\sim$19 expose: a trilogy of goal-discovery negatives.}
No reward is found on the rest even at $25\text{k}\times4$-seed interact-biased search: their wins are
not \emph{reachable}, they must be \emph{understood}. We ran three increasingly sophisticated,
principled attacks---all planning \emph{through} the code world model---and report them honestly:
(i)~\textbf{E102}, a core-knowledge perceptor generating atomic candidate-goal \texttt{CodeObjective}s
(reach/cover/collect/extremize): \textbf{0/9}; (ii)~\textbf{E103}, a frontier model (Claude)
\emph{generating} rich, \emph{compositional}, closed-loop-refined goal hypotheses: \textbf{0/3};
(iii)~\textbf{E104}, procedure-aware Bayesian active inference over composite \emph{subworlds} with
TROPICAL-semiring hierarchical planning: \textbf{0/3}. None succeed \emph{even with a fidelity-$1.0$
model}. The trilogy yields a sharp diagnosis: the walls are \textbf{not} atomic goals, and the
failure is \emph{representational}---many ARC-AGI-3 wins are a \textbf{goal-as-procedure} (a specific
ordered protocol), which a goal-score over a single state cannot express, so planning optimizes the
wrong target (this is precisely why undirected \emph{search} stumbles into some wins while
goal-conditioned \emph{planning} finds none).

\paragraph{A visual perceptor foreshadows the fix (E106).} Rendered in the official palette and read
by a \texttt{VisionPerceptor} (Claude-vision), the frames yield \emph{semantic} goal hypotheses the
structured methods miss---e.g.\ \texttt{ka59} as ``two rooms joined by a corridor, a purple door, a
green agent, a target: cross the door.'' We did not turn this into a solver, but it points to the
resolution: the procedural walls need a \emph{reasoning} reader of the game, realized in
\S\ref{sec:agent} as a live coding agent. (We note this as a bridge, not a result.)

\paragraph{Search vs.\ reasoning.} The automated methods expose a clean dichotomy: directed search and
multi-perception consensus solve the \emph{reachable} games, but a comprehensive, principled
goal-discovery toolkit (atomic objectives, LLM hypotheses, Bayesian sub-worlds) does \emph{not} close
the goal-as-procedure walls---which is precisely the boundary a \emph{reasoning} agent crosses
(\S\ref{sec:agent}). Leaderboard comparisons and the offline-replay caveat are stated precisely in
\S\ref{sec:agent}.

\section{Multi-perception consensus world models (the winning method)}
\label{app:consensus}
The method that moves the needle perceives each game through \emph{multiple modalities} and
synthesizes their discovered world models by consensus---all in the \openworld{} structure
(Fig.~\ref{fig:arch}).

\centering

\textbf{Two senses of ``perception.''} A clarification, since the word is overloaded. In this section
``modality'' means \emph{action modality}---the two interaction channels ARC-AGI-3 uses (directional keys
vs.\ coordinate clicks)---and the board state is read by a \emph{single} perceptor (the masked frame). The
consensus below therefore selects across \emph{action channels}, not across \emph{perceptual views}. Fusing
several perceptual \emph{lenses} of the \emph{same} frame is a distinct, orthogonal axis, taken up under
``Two axes'' below.

\textbf{Perception.} ARC-AGI-3 mixes two action modalities: directional keys
(\texttt{ACTION1--5,7}) and \emph{coordinate clicks} (\texttt{ACTION6} with $(x,y)$). Many games are
click-only (e.g.\ \texttt{vc33}); the board responds only to clicks on valid sprite cells, which we
infer \emph{from pixels} (small connected components + rare colors). A UI step-counter changes every
frame, so we detect and \emph{mask} those cells before hashing---otherwise the state graph never
compresses. The masked-frame signature is the perceptor's state.

\textbf{Discovered map = an \openworld{} \texttt{World}.} Frontier-biased rollouts build the reachable
state-transition graph at \emph{world-time compute}---by which we mean inference-time search that
\emph{constructs the world model during play} (test-time compute spent building the model and its map,
not sampling outputs). The discovered $(s,a)\!\to\!s'$ table becomes a \texttt{World} via
\texttt{FunctionTransition}, with \texttt{levels\_completed} (or an induced \texttt{CodeObjective}) as
reward. \texttt{to\_spec} renders its \texttt{preview.graph}---the
game's \emph{map}---as an atlas card, viewable in \texttt{openworld serve /view}.

\textbf{Synthesis = consensus hard-voting across modalities.} Each modality yields its own discovered
world model; we combine them with \texttt{ConsensusTransition} (select the highest-fidelity modality,
or per-state vote). \emph{Ablation:} of the $10$ games this tier solves, the directional modality
accounts for $7$ and click for $3$---neither single modality reaches all $10$, so combining modalities
adds coverage (\texttt{ar25} needs directional, \texttt{vc33} click). We are candid that this is a
\emph{selection across views}, not a deep mechanism: its value is breadth, and the genuinely
surprising case (\texttt{s5i5}, solved here after defeating the goal-discovery trilogy on a perfect
model) is the one worth studying further.

\textbf{Two axes of ``multi,'' not one.} The consensus above runs along the \emph{action-modality} axis
(keys vs.\ clicks), with the state read by one perceptor (the masked frame). A second, orthogonal axis is
\emph{perceptual perspective}: fusing several \emph{lenses} that read the same frame differently---object
configuration, salience, the step-counter as a \emph{value} rather than a masked nuisance, symmetry, color
algebra, coarse regions---and selecting the highest-fidelity \emph{view} per state. The two are
complementary: multi-modality adds \emph{interaction} coverage (some games respond only to clicks), while
multi-perspective adds \emph{representational} coverage (some wins hinge on a feature a single lens aliases
away). This tier exploits only the first axis; extending consensus along the perceptual-lens axis is the
natural complement, and it is where the reasoning agent---which writes and re-perceives its own world
model (\S\ref{sec:agent})---takes over the goal-as-procedure walls this tier cannot crack.

\textbf{Result.} Multi-perception consensus solves \textbf{10/25} public games (Fig.~\ref{fig:bar}),
including four---\texttt{cn04, lf52, m0r0, s5i5}---reached by \emph{no} other method; the union of
the \emph{automated} strategies is \textbf{12/25} (the live coding agent then lifts the total to
$\geq$1 level on all \ArcUnionGames{}). \texttt{s5i5} is the headline: it had resisted the entire
goal-discovery trilogy on a perfect model, yet world-time-compute exploration over the right
perceptual modality completes it. The takeaway for world-model research: \emph{the leverage is in
perception and synthesis, not in a cleverer single planner.} What it does \emph{not} crack are the
goal-as-procedure walls---which is exactly where a reasoning agent that writes its own world model
takes over.

\section{A routed solver: 25/25 as one pipeline, not a union}
\label{sec:router}
The $25/25$ above is a \emph{union} across strategies---run everything, keep the best---which is
post-hoc. It is naturally realized as a deployable \textbf{cascade}: run the cheap multi-perception
solver under a fixed budget; if it completes a level the game was \emph{reachable} and we stop,
otherwise \emph{escalate} to the live coding agent. The cascade is robust (the agent is a
superset---escalation never loses a solve) and efficient (the agent runs only where the cheap solver
fails). It induces a clean partition: \textbf{12} games solved by the cheap tier, \textbf{13}
escalated to the agent. We report this partition from the banked, replay-verified \emph{tier results}
(equivalently, the cascade's outcome); we did \emph{not} separately run a faster upfront reachability
classifier end-to-end---a natural optimization (a small search probe should predict the same split,
since walls never fire a reward) that we leave to future work. We present this honestly as a simple
\emph{try-cheap-then-escalate} dispatch, not a novel routing algorithm; its only claim is that it
turns the union into one pipeline that invokes the expensive reasoning agent on barely half the suite.

\section{On-demand goal synthesis: a Bayesian SLM rung (E119)}
\label{sec:e119}
The goal-inference wall (\S\ref{sec:solvewall}) showed that scoring a static \emph{state} cannot express an
ordered \emph{procedure}. A complementary response, drawn from the Model Synthesis Architecture
(MSA; Wong et al., \texttt{arXiv:2507.12547}), is to treat the goal itself as something a model
\emph{synthesizes on demand} rather than scores: a small language model (SLM) emits the level's
objective as an executable \textbf{predicate-program} ($\texttt{reach}\,/\,\texttt{count}\,/\,\texttt{align}$),
and we keep a \textbf{posterior over these programs}---sample best-of-$N$, cluster by \emph{observed
behavioral effect} on probed frames, weight by cluster mass, and \textbf{abstain} when no hypothesis
clears a threshold~$\tau$ (Fig.~\ref{fig:e119}). The synthesized program is used \emph{only} to order an
env-ground-truth search (a \texttt{score\_fn} priority); correctness comes entirely from deterministic
replay, so the SLM can make search \emph{faster} but never \emph{wrong}. The invariant gives a free
control: a \emph{search-only} rung (no SLM) lower-bounds the \emph{slm} rung over the identical
reachable set, isolating ``does on-demand model synthesis speed search?'' as the measured effect. The
solved path is emitted as an \openworld{} \texttt{World} (masked-frame state, a \texttt{FunctionTransition}
over the learned $(s,a)\!\to\!s'$ table, an induced \texttt{CodeObjective}), so the Bayesian rung
inhabits the same structure as every other method here. We present E119 as a \emph{design with a verified
control rung}, not a solved-count claim: the deterministic core and abstaining proposer are implemented
and unit-tested; the search-only control banks $0$ levels on the directional pilot games (blind BFS does
not crack procedure goals, as expected), and the SLM-in-loop measurement is left as the immediate next
run.

\begin{figure}[t]
\centering
\includegraphics[width=\textwidth]{figs/e119_msa_strategy.png}
\caption{\textbf{The MSA Bayesian loop inside an \openworld{} solver (E119).} \openworld{} primitives are

masked-frame state $\sigma(s)$ and a \emph{relational} \texttt{object\_json} scene (the only perceptor
that conditions the proposer). The \textbf{Bayesian subgoal} synthesizes $N$ goal-programs
$\pi_i\!\sim\!p(\pi\mid\texttt{object\_json})$, forms a posterior by clustering on behavioral effect
(cluster mass $\approx\hat p(\pi\mid\text{obs})$), and either commits the $\arg\max$ program $\pi^*$ once

env-ground-truth search; a child that raises \texttt{levels\_completed} is a replay-verified solve, which
is banked and emitted as an \openworld{} \texttt{World} map. \textbf{Invariant:} the synthesized model

\section{Multi-world dynamics: voting experts and a self-rebuilding world model (E120--E123)}
\label{sec:selfrebuild}
E119 keeps a posterior over \emph{goal-programs}; the complementary object is a posterior---and a
structural response---over \emph{dynamics}. Two pressures motivate it. First, when repeated attempts on a
game \emph{plateau}, the agent has tunnelled on one framing of the mechanic; a single proposer cannot
diversify itself. Second, ARC-AGI-3's compositionality cliff means the rules literally \emph{change} at
level boundaries, so any one fixed transition model is wrong past the next level. Both are answered by
treating the world model as \emph{plural and revisable}, using only primitives already in the framework.

\textbf{E120 --- a committee of expert worlds.} A stuck game is routed to a panel of differently-primed
coding agents---one \emph{lens} each (topology, temporal/phase, force-transmission, color-algebra,
symmetry, perception-reframe, counting)---so the diversity is in \emph{framing}, not model size; the panel
\emph{replaces} the repeated identical prompt, so it is cost-neutral. Each lens that produces a
replay-verified trace is reconstructed into its own \openworld{} \texttt{World} (a \texttt{FunctionTransition}
over that run's discovered $(s,a)\!\to\!s'$ table), and the lenses are combined with
\texttt{ConsensusTransition}$(\text{mode}{=}\texttt{vote})$: a per-state majority over the members'
predicted next-states, each weighted by its env-verified levels reached (Fig.~\ref{fig:selfrebuild}A). A
shared per-game status-mask and a global state-id map make identical physical states vote on the same
node, so the vote is well-posed. The frames are thus genuine \openworld{} worlds, not prompts: $N$ experts
$\to$ $N$ worlds $\to$ one consensus world.

\textbf{E121 --- a world model that notices when the rules change.} Replaying a verified trace, we
online-learn the dynamics and watch the model's own \textbf{surprise}: an anomalously large masked-board
delta---a level-up reloads the board---signals that the rules just changed. We segment the trace at those
surprise spikes, build a per-regime world model, and compose them forward with \texttt{PhasedTransition},
whose dynamics advance \emph{irreversibly} and record the active regime in state (Fig.~\ref{fig:selfrebuild}C):
a single \openworld{} \texttt{World} that rebuilds its own dynamics at each detected change. The segmenter
the solver uses takes \emph{every observable signal}: an observed level-up (\texttt{levels\_completed}
increments---the solver's own reward, seen each step) \textbf{union} a board-surprise spike. Because every
rule change either bumps the level counter or is caught by surprise, this is \textbf{complete: recall $1.0$
on all $25$ games} (Fig.~\ref{fig:selfrebuild}D). Surprise then \emph{adds} within-level regime shifts the
counter alone would miss. As an \emph{ablation}, we ask how much is visible \emph{from frames alone}---the
board-delta cue with the level signal withheld: it recovers a mean of $0.84$ (median $1.0$, $16/25$ perfect;
\texttt{ka59} all six, precision $0.857$, Fig.~\ref{fig:selfrebuild}B), failing only where a level-up does
\emph{not} reload the board---incremental ``build'' games like \texttt{cd82} and \texttt{bp35}, whose rule
changes the level counter still captures. This is the MSA on-demand-synthesis loop (\texttt{arXiv:2507.12547})
realized over \emph{dynamics}: surprise (plus the reward) is the trigger to synthesize a new sub-world and
carry it forward.

\textbf{E122/E123 --- the loop made online.} Two further steps close it. An \emph{online} monitor (E122)
runs the same segmenter \emph{causally}, from signals available each step: a running baseline of the
in-regime board delta (warmup $+$ refractory gap) union the observed level-up. Combined, it is again
complete---\textbf{recall $1.0$ on all $25$ games}, online; the frames-alone ablation recovers a causal mean
of $0.74$ (strictly harder than retrospective, under-firing on very short traces). The reset itself (E123)
respects the replay-only env: on a boundary we \textbf{replay from \texttt{reset()} to the last completed
frame} (a fresh env, since \texttt{reset()} pollutes a progressed game) and build a \emph{fresh, independent}
model for the new regime, appending it to the \texttt{PhasedTransition} chain while earlier regimes stay
frozen. This mechanism is \textbf{robust across all 25 games}: the replay lands on the exact boundary frame
for \emph{every} regime ($25/25$ games), and the resynthesized composite reproduces play \emph{exactly}---%
round-trip fidelity $\mathbf{1.0}$ on all $25$. What remains open is therefore not the re-synthesis
mechanism but the autonomous deep \emph{search} that would discover the actions in the first place (the
goal-inference wall of \S\ref{sec:solvewall}); here exploration is trace-guided to isolate the
mechanism. The same surprise-driven reset is wired into the live expert panel as a prompt instruction
(reject a stale model rather than the level---a wall is often a stale model, not an impossible level).

\section{Codex-steered autonomous deep search (E124, design)}
\label{sec:e124}
Every method above is, structurally, an \openworld{} world model: a \textbf{perceptor} turns the replay-only
frame into a discrete state $\sigma(s)$, the agent's exploration builds the $(s,a)\!\to\!s'$
\texttt{FunctionTransition} \textbf{dynamics}, an \textbf{objective} scores progress, and the result
serializes (\textbf{emit}) to a map and atlas card. The goal-inference wall (\S\ref{sec:solvewall}) is
precisely a missing \emph{objective}: a static state score cannot express an ordered \emph{procedure}, so
blind search banks $0$. E124's design keeps the world model intact and swaps only the objective's
\emph{source}. A strong external coder (OpenAI Codex) compiles the goal \emph{source-free}---as an ordered
list of subgoal predicates plus macro-actions (multi-step options that collapse search depth)---and a
best-first search uses them to \emph{order} exploration while the environment \emph{decides} correctness (a
replay-verified \texttt{levels\_completed} bump). On a level-up the surprise monitor (E122) fires, the world
replays to the boundary and rebuilds (E123), and Codex re-compiles the goal for the new regime; competing
hypotheses combine by \texttt{ConsensusTransition} (E120). Figure~\ref{fig:e124flow} traces how each stage
fills an \openworld{} world slot. As with E119, we present this as a \emph{design with a fair control}: the
measured claim is whether Codex-ordered search beats an \emph{identical} blind-BFS loop on the same budget
and reachable set; every Codex call is source-free-audited (it never reads a game's code) and captured for
deterministic replay. The build is in progress and gated on that lift.

\section{Primer: what is a ``world'' in a world model?}
\label{app:primer}
Readers often ask what a ``world'' is, and how a \emph{world model} differs from the world. They are
the \emph{same kind of object} (Fig.~\ref{fig:anatomy}). A \textbf{world} is an environment described
by five parts: a \emph{state} $s$, a set of \emph{actions} $a$, a \emph{transition} $s'=T(s,a)$ (how
the state changes), a \emph{perception} boundary (observation $\to$ state), and a \emph{reward/emit}
boundary (state $\to$ outcomes/outputs). A \textbf{world model} is simply a \texttt{World} object that
\emph{mirrors} an environment-world with the same five parts; \emph{modeling} means building a
\texttt{World} whose transition reproduces the environment's (here, exactly, as verified code). So
``the model of a world is itself a world,'' and worlds \emph{nest}---a \texttt{CompositeWorld} contains
sub-\texttt{World}s (e.g.\ a movement subworld and a counter subworld): the ``world in a world.''

\begin{figure}[h]
\centering
\includegraphics[width=\textwidth]{figs/arc3_anatomy.png}
\caption{\textbf{Anatomy of a world model.} The environment (the game) is a world; our model is a
\texttt{World} with the same anatomy---perception $\to$ state $\to$ transition, with actions and
reward---and worlds nest (\texttt{CompositeWorld}). Bottom: the concrete ARC-AGI-3 mapping.}
\label{fig:anatomy}
\end{figure}

In OpenWorld a \texttt{World} is first-class. For ARC-AGI-3 the mapping is concrete: state $=$ the
status-masked $64\times64$ grid; actions $=$ \texttt{ACTION1--7} (directional) or
\texttt{ACTION6}$(x,y)$ (click); transition $=$ a verified \texttt{predict(frame,action)}
(\texttt{CodeTransition}); perceptor $=$ the masked-frame / object-graph reader; reward $=$
\texttt{levels\_completed} (or an induced \texttt{CodeObjective}). Our \emph{discovered} world model is
the explored $(s,a)\!\to\!s'$ table wrapped as a \texttt{FunctionTransition}; \texttt{to\_spec} renders
its reachable-state graph as the viewable ``map.''

\begin{lstlisting}[style=housespecyaml, language=Python, caption={The discovered \texttt{ka59} world model is an ordinary \texttt{World}.}]

    return {"sig": table.get(state["sig"], {}).get(action.name, state["sig"])}
world = O.World(name="arc3-map-ka59",
                initial_state={"sig": s0}, actions=actions,
                transition=O.FunctionTransition(step))
spec = O.to_spec(world)        # spec["preview"]["graph"] is the reachable-state MAP
O.render_card(world, "ka59.svg")   # the atlas card; serve it at /view
\end{lstlisting}

\section{The live coding agent: harness and protocol}
\label{app:agent}
The live coding agent is Claude Code (\texttt{claude -p}, one agentic session per game, no repeated
sampling) given a fixed harness and an unbounded \emph{offline} budget under the deterministic env.
\textbf{Leakage:} the prompt supplies only the harness API and the generic recipe---it does \emph{not}
contain the win condition, which the agent must infer. \textbf{Harness:} \texttt{Game(gid)} exposes
\texttt{reset()}, \texttt{step(a)} / \texttt{step(6,x,y)}, and \texttt{frame/levels/win/avail};
success is any increase in \texttt{levels}; determinism lets the agent search offline then replay-verify.
The agent writes/edits Python (explore $\to$ synthesize a \texttt{predict()} world model $\to$ reason
the goal $\to$ plan $\to$ replay-verify) and saves the winning action list to \texttt{solved.json};
we then re-verify it independently against the live game. Full launcher: \texttt{scripts/run\_arc\_agent.sh}.

\section{Solved-level replay trajectories}
\label{app:replay}
\paragraph{What these figures show (in plain terms).} ARC-AGI-3 is a set of tiny video-game puzzles. A
player sees only the screen---a $64{\times}64$ grid of colored squares---and a few buttons (arrow keys,
and on some games a mouse click), and is told \emph{nothing}: no rules and no goal in words. It must work
out, purely by pressing buttons and watching how the screen changes, how to finish each of the game's
levels. As our solver explores, it builds a \emph{world model}: an internal map of how the game works
---which distinct \emph{screens} the game can show, and which \emph{button} moves it from one screen to the
next. That map is the whole point of the paper: once you have it you can \emph{plan} a route to the goal
instead of stumbling around. For each of the eight games the solver finished completely, the top of the
figure is the map it built and the bottom is a \emph{replay}---real screenshots of the game at the start
and at the instant each level is cleared, played back from the exact sequence of button presses the solver
found (which we re-ran against the real game to confirm). Read left to right, the replay shows the game
visibly transforming as level after level is cleared. Figures regenerate from
\texttt{scripts/make\_arc3\_replay\_figs.py}.

\paragraph{How to read the map (top panel).} Think of it as a subway map of the game.
\begin{itemize}\setlength{\itemsep}{1pt}
  \item Each \textbf{box} (\texttt{q0}, \texttt{q1}, \dots) is one \emph{distinct screen} the solver
  found. The solver cannot store every pixel, so it boils each screen down to a fingerprint and gives
  identical fingerprints the same name; \texttt{q0,q1,\dots} are therefore just labels---``screen~A,
  screen~B''---and neither the \emph{number} nor a box's \emph{position} carries meaning (the layout is
  arranged automatically for readability).
  \item Each \textbf{arrow} (\texttt{s1}, \texttt{s2}, \dots) is a \emph{button}: an arrow from one screen
  to another reads ``pressing this button on this screen takes you to that screen.'' The \texttt{start} tag
  marks the screen the game begins on.
  \item The small print at the bottom of the card (\textsc{state schema}, \textsc{actions},
  \textsc{dynamics}, \textsc{sample rollout}) is the same world model written out as data: the one
  fingerprint variable, the button vocabulary, the learned look-up table that powers the arrows, and a tiny
  preview. A card showing just one self-looping box means the fingerprints collapsed the explored region to
  a single screen---the solution still comes from the verified button sequence, not the picture.
\end{itemize}

\paragraph{How to read the replay (bottom strip).} Each panel is an actual game screenshot in the ARC
16-color palette. The first is the starting screen; each later one is the board at the moment a level is
completed (labeled with the move number that did it). The scattered colored blocks are the game's real
pieces---walls, keys, goal markers---not noise.
