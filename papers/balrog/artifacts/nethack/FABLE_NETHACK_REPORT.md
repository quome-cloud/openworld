# FABLE_NETHACK_REPORT — world-model synthesis + classical search on BALROG NetHack (full NLE)

**Synthesis model:** Fable 5 (max reasoning). Runtime is LLM-free pure code (no API calls at test time; audit in §2).
**Claim under test:** the world-model-synthesis + classical-search recipe (Baba Is AI 100%, MiniHack 92.5%) extends to BALROG's unsolved column: full NetHack (NetHackChallenge-v0).
**Benchmark protocol:** BALROG `nle` config verbatim — NetHackChallenge-v0, 5 episodes, `character '@'` (random role), `max_episode_steps 100_000`, `no_progress_timeout 150`, `skip_more True`. Progression = BALROG's achievement curve: max over reached `Dlvl:n` / `Xp:n` milestones (from `balrog/environments/nle/{progress.py,achievements.json}` — e.g. Dlvl:5→0.026, Dlvl:8→0.070, Dlvl:10→0.126, Dlvl:13→0.257; Xp:8→0.075, Xp:10→0.179); episode score = final progression, suite score = mean × 100. Read at episode end via `env.get_stats()`, the same call BALROG's evaluator makes.
**SOTA baseline (raw-HTML parse, 2026-07-06):** **6.8 ± 3.2 (Gemini-3-Pro)**; runners-up 4.0 ± 0.8 (Gemini-3-Flash), 3.0 ± 0.5 (Gemini-3.1-Pro). Evidence: `artifacts/nethack/evidence/balrog_leaderboard_2026-07-06.html` + `leaderboard_parse.json`, generated by `verify_leaderboard.py` (curl + regex over the leaderboard-table `<td>` cells; no summarizer in the loop — a summarizer misread of this table cost this arm its original calibration: 35.0 was the MiniHack column).

## 1. Method

Same skeleton as the Baba and MiniHack arms: read engine/wrapper sources **offline** → synthesize a task-scoped belief-state world model → plan with classical search per step → verify closed-loop through the message channel. NetHack forced three escalations beyond the MiniHack arm:

1. **Objective model first.** BALROG's NetHack metric is not "win" or "score": it is a monotone achievement curve over dungeon depth and experience level, dominated by depth (Dlvl:10 = 0.126 ≈ 2× Dlvl:8; Xp milestones lag Dlvl at equal effort until Xp:9+). Survival per se scores nothing. The synthesized policy is therefore a **depth-before-death maximizer**: descend as fast as the survival layers allow, treat death after new depth as an acceptable cost. Two concrete consequences validated in dev: (a) an XP-pacing gate ("grind until xplvl supports the depth") *hurt* — the dig-diver it stopped died grinding at depth 7 where ungated it reached 10; (b) resting/prayer/food layers pay for themselves only as descent enablers, tuned to spend turns, not to avoid risk.

2. **Multi-level belief state (Atlas).** One persistent `LevelMap` per `(dungeon_number, level_number)` from served `blstats`: monotone glyph-integrated terrain, dark-cell negative inference at radius 1 (MiniHack engine lesson, suppressed while blind), stairs/holes caches, per-level suspect-wall sets from failed-move detection (aged every 250 steps — stale suspects walled off corridors in dev), boulder-push failure memory, peaceful-monster no-attack cells. Terrain under the agent is invisible (engine lesson: cell tops only) — solved with a **`look` protocol** on every level arrival (dev seed 102 dug a 500-step loop on top of unrecognized stairs before this existed).

3. **Layered policy, replanned every step** (`nh_agent.py`):
   P0 prompt/menu state machine (served `misc` flags + message grammar + repeat-escape ladder) → P1 scripted-sequence queue (dig / eat / pray / kick / open / pickup, each message-verified, aborted on "You don't have that object"/"Never mind") → P2 engulfed-attack → P2.5 `look` on arrival → P3 survival (prayer in major trouble only, with anti-wrath discipline + last-resort gamble; crisis flee only when it fully disengages and every adjacent threat is slower than speed-12; held/pit states force-fight) → P4 food clock (inventory food → fresh safe-corpse (offline species table, race-aware cannibalism guard) → prayer) → P5 combat (weakest-first melee, never-melee species table, immobile-monster bypass, door-diagonal legality) → P6 rest gate (HP<60% → search-rest to 85%, budgeted per level, only with no mobile hostile visible) → P7 descent: **dig straight down when a digger is held** (Archeologist start, or any pick-axe/mattock spotted by object glyph and detoured to ≤20 cells, with a threat gate: clear hostiles within 3 before starting) else stairs/trapdoor pathing (BFS, doors opened/kicked, bad-trap avoidance with two-pass relaxation) → P8 digger acquisition → P9 mass-biased frontier exploration with target persistence → P10 doors → P11 hidden-passage search rotation (dead-end bonus) → P12 anti-no-progress fallback (`search` always advances game time; the 150-step no-progress abort never fired in any logged episode).

The dig-dive is the recipe's sharpest expression here: an Archeologist fell Dlvl 1→10 in **102 env steps** (~7 steps/level: apply → letter → `>` per level, plus interruptions), where stairs-seeking roles need 100–1300 steps/level. Digging turns the descent problem from search into arithmetic.

**Offline provenance (disclosed):** `nh_common.py` embeds lookup tables computed at import time from the NLE python API (permonst species name/level/difficulty/speed, cmap symbol semantics, object glyph ids for pick-axe/mattock, safe-corpse and never-melee species lists distilled from NetHack source knowledge). These are game-rule constants keyed by served glyph ids — the disclosed "world model from source" leg of this research program. The quarantined source-blind induction leg receives only `results/transitions/` (§6).

**Interface findings** (properties of BALROG's NLE stack that shape any agent, discovered clean):
- BALROG's NLE action list (248 strings) contains every letter a–zA–Z (as raw-key aliases of commands: "d"→drop→'d' at prompts) and digits, but **no `-` (minus)**: the "write with" getobj prompt cannot select fingers, so **fingertip-Elbereth — the canonical NetHack panic button — is unreachable** for every BALROG agent. Survival layers had to be built around prayer/flee/rest instead.
- `NetHackChallenge` allows all modes: menus stay open (unlike MiniHack's auto-SPACE), so menu text on `tty_chars` is parseable and Escape-closable; `--More--` is auto-answered by BALROG's AutoMore wrapper which also accumulates multi-part messages into one string per step (a better sensor than raw NLE).
- `reset(seed)` is honored by the BALROG fork (NLE's anti-seeding guard is dead code there): episodes are exactly reproducible, role included. Fixed seed blocks chosen in advance: dev 101–140, condition A 1000–1004, memory 4000+/5000+/6000+.
- Zero-time actions (prompt keys, bumps) freeze the world exactly as in MiniHack; the env's only step-economy constraint is the 150-env-step no-progress abort, which a `search` fallback trivially clears.

### The metric, and why this policy shape is rational

BALROG's NetHack progression is a **max-rung ladder**: the episode scores the highest achievement rung ever touched (Dlvl:n or Xp:n, whichever maps higher on the curve), not a sum, not survival time. Three consequences drive everything above: (1) *depth-diving is rational* — each new floor is a permanent, un-losable rung, and the rungs grow superlinearly (Dlvl:10 = 2× Dlvl:8), so trading survival odds for floors is +EV until death is immediate; (2) *post-milestone deaths are free* — a character that dies one step after reaching Dlvl:10 scores identically to one that retires there; (3) *variance is structural* — the score distribution is a race between a geometric survival process (each floor is a fresh chance to meet a giant spider) and heavy-tailed role/dungeon luck (Archeologist pick-axes, early trapdoors, Mines dwarves), so per-episode scores span 0–31 and 5-episode means swing by ±3.

## 4. Memory experiment (condition B)

Hypothesis (operator): NetHack is the canonical die-and-learn game — cross-episode memory should pay where the baseline leaves attempts on the table.

**Result: not supported — again, and more decisively than in MiniHack.** 3 passes × 5 episodes (fresh seed blocks 4000/5000/6000, ledger accumulated only from these episodes' own logged observations):

| pass | score | per-episode prog% |
|---|---|---|
| B1 | 5.97 | 12.6, 2.4, 9.8, 3.5, 1.5 |
| B2 | 7.18 | 20.6, 2.1, 9.8, 1.8, 1.5 |
| B3 | 4.37 | 1.5, 1.5, 2.7, 0.0, 16.1 |
| A-regime reference (untouched 25-ep block, §3) | 4.39 | — |

No learning curve (5.97 → 7.18 → 4.37); the pass-3 dip is the memory itself misfiring:
- **M1 (avoid-species) is anti-productive at this skill level.** The species that killed passes 1–2 characters were *trash mobs* — newts (2 deaths), goblins (3), bats, jaguars — because weak roles die to trash. M1 then promoted them to never-melee for xplvl ≤ 6 characters, so pass-3 agents refused to fight the trash mobs in their path... and were killed by them anyway while stepping away (pass-3 ep0: Healer killed by a newt on Dlvl:2 with `newt` on the loaded avoid-list). Avoidance learned from deaths selects exactly the monsters you cannot avoid.
- **M2 (danger-depth rest gate)** fired correctly after the ledger fix (combat deaths only) but its firings (70+ in one episode) mostly slowed descent; with depth-9–12 deaths dominating the ledger, the tightened gate never prevented an observed death mode (deaths came from speed-15+ monsters where resting earlier does not help).
- The v1 ledger run (killed and restarted, disclosed) had two bugs worth recording as memory-design lessons: case-sensitive death-cause parsing (all causes null), and a depth-1 *starvation* death setting danger-depth=1 — a pacing rule keyed on a non-combat death tightened the rest gate everywhere. Provenance-cited memory is necessary but not sufficient; the *semantics* of what a death teaches need modeling.
- Ledger artifacts: "killed by an orcish dagger" and "poisoned by a rotted large kobold corpse" were recorded as species — death-cause strings name proximate causes, not agents.

The MiniHack arm concluded memory needs headroom (attempts left on the table) to pay. NetHack has the headroom, and memory still didn't pay — because the knowledge that transfers across NetHackChallenge episodes (species stats, death depths) is exactly the knowledge the offline world model already encodes better (permonst difficulty/speed tables). What die-and-learn *would* need here is knowledge the offline model cannot have: per-game item identities, per-level hazards — which do not transfer across fresh-seed episodes by construction. **The memory hypothesis fails on NetHackChallenge not for lack of headroom but because the episode boundary destroys the transferable state.** A paired same-seed memory run (B2 design, below) tests the remaining channel.

Design (fixed before running): persistent ledger (`nh_memory.py`) accumulated only from condition-B episodes' own logs, every entry citing its transition file. ## 6. Diagnosed-failure catalog (deliverables in their own right)

Every one of these was found through logged clean observations only, and each is a general property any BALROG NetHack agent must handle:

1. **Pet-boxing.** A pet follows the agent; if pets are path obstacles, the planner sees zero legal moves at any dead-end the pet follows you into — thousands of stationary searches. A pet is a door that swaps with you, not a wall.
2. **The wrath chain.** Cannibalism (race-string mismatch: "dwarven" ≠ "dwarf"; "dwarf zombie corpse" passing a substring check) → god anger → every later prayer is a coin-flip → "killed by the wrath of X" epidemic. Food-safety needs exact species matching plus race awareness; prayer needs budget discipline (T>700 first, 1500-turn gaps, ≤3, one last-resort gamble).
3. **Zero-time livelocks.** A refused `eat` (phantom corpse — kills don't always drop one, and the corpse drops on the victim's cell, not the attacker's) consumes an env step but no game time, so time-keyed caches never expire and the env's 150-step no-progress abort fires. Every zero-time loop needs a game-time-advancing breaker.
4. **Never-melee blockers.** A sleeping gas spore + acid blob bracketing a 1-wide corridor walled off a level for 4,000 turns: refusing melee is right, but the agent needs a ranged answer (throw layer) and an unreachable-candidate-skipping search rotation.
5. **Speed arithmetic in flee decisions.** "Slower than us" is not enough: a speed-9 crocodile still attacks on ~3/4 of turns while "outrun" at 12, and direction flip-flop gains zero distance. Flee needs a real speed margin (≤2/3), sticky direction, and full disengagement (0 adjacent after the move) — else fight.
6. **Early Mines vs weak roles.** Armed dwarves are the dominant killer of xplvl-1–2 characters; branch choice is a survival decision (avoid Mines until xplvl 4, with a bounded give-up).
7. **Metric-driven descent.** The BALROG metric pays depth-before-death; pacing gates that trade descent for safety (XP grinding) measurably lower the score. Optimal play here is not optimal NetHack.
8. **Stale-door churn (found by the robustness block, unfixed by design).** An item lying on an opened door keeps the remembered terrain "closed" (the item glyph covers the door cmap); the door layer then loops `open`→direction→"This door is already open." — 2 env steps, ~0 game time — and the churn advances time just often enough (~1 turn / 40 steps) to evade both the env's 150-step no-progress abort and the agent's own livelock breaker. Robustness ep1 burned its 100k-step budget on Dlvl:1 this way. The fix (message-driven terrain correction: "already open" ⇒ rewrite cell to open) is one line, but it was found *after* the scored blocks ran, and the robustness guard is only meaningful on byte-identical agent code — so it is documented rather than applied. This is the exact failure mode the guard exists to surface.

## 7. Limitations (reviewer-grade)

1. **Leaderboard comparability.** BALROG's leaderboard evaluates LLMs choosing actions by generation; our runtime is pure code acting through the same interface (a claim about the synthesis+search recipe, not a leaderboard submission). Same asterisk as the Baba and MiniHack arms.
2. **No SOTA-beat claim survives n=25.** The powered baseline is 4.39 [2.97, 5.97] — below Gemini-3-Pro's 6.8 point estimate, statistically indistinguishable from it (their ±3.2 interval overlaps ours), indistinguishable from #2 (4.0 ± 0.8), above #3's midpoint. The n=5 official-protocol 7.64 "beat" is reported as a protocol artifact, not a result. Symmetrically, the leaderboard's own n=5 NetHack entries carry the same noise and their ordering should not be trusted.
3. **Offline source-derived model.** Terrain/species/objective tables were synthesized from NLE sources offline (disclosed provenance); at test time nothing flows from env internals. The induction leg (§5) is the removal experiment.
4. **Official-seed iteration.** Two runs against the official block (7.07 → 7.64) with dev-validated fixes between; dev seeds (101–140) and the robustness block (2000+) are disjoint from the official block (1000+).
5. **Env-version pinning.** balrog-nle 0.9.0 + vendored BALROG stack; the seedability of NetHackChallenge (dead anti-seeding guard) and the auto-More message accumulation are properties of this fork.
6. **Unmodeled subsystems.** Ranged enemy attacks (ep3's wand death), armor/AC economy, spellcasting/ranged offense (beyond the never-melee throw layer), altars/shops/Sokoban, hidden-passage posteriors. Each is a known lever the next iteration can add; the report's failure catalog quantifies what they cost.
