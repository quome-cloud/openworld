Motivation: from computing to world computing

What is a world computer?

  • Hardware + software that instantiates, verifies, composes, runs simulated worlds
  • Dynamics are executable, checkable code -- not learned weights
  • Classical computing made computation a cheap, inspectable commodity
  • World computing aims to do the same for worlds
  • Unit of work: a world model -- predicts next state given state and action

Two species of world models

  • Perceptual/generative: neural nets over pixels/latents (Dreamer, MuZero, Genie, Sora)
  • They are data-hungry, compounding, and opaque
  • Code World Models (CWM): transition is a human-readable program over symbolic state
  • OpenWorld writes down what can be told, and verifies it
  • Complementary tools, not competitors

Why verified code

  • Value is not novel dynamics -- the model can often recite the rule
  • It is exact, fast, auditable execution an agent can call as a tool
  • Deterministic: no compounding error; learned dynamics decay as (1-e)^t
  • Plain code rolls out orders of magnitude faster than a per-step LLM
  • PyTorch didn't give new models -- it made them fast to prototype

World, action, agent, simulation

  • World = symbolic state + discrete actions + verified transition
  • Agent = a separate policy; a user of a world, not part of it
  • Simulation = agents in a world producing scored trajectories
  • A verified model relocates the sim-to-real gap into spec + perception
  • So one World serves as both ground-truth environment and planning model

The world computer, benchmarked on cost, latency, performance

One agent authors validated, servable worlds; verified code rolls out ~47,000x faster than an LLM
  • A: 100 worlds authored at a median of 2.8 min each (p90 3.9)
  • B: verified code at 14,997 steps/s vs 0.32 for a per-step LLM proxy
  • C: 100% validate structurally, 99% pass a behavioral execution audit

The substrate: system design

Six unit-testable components

  • World: state ontology, actions, rules, pluggable transition engine
  • Three engines: CodeTransition, LLMTransition (baseline), FunctionTransition (oracle)
  • Agent: LLM planner or deterministic policy
  • Objective + Dial: open, tunable value specification
  • Simulation, Tuner, Judge -- all over a local Ollama backbone, zero runtime deps

The plan--generate--verify relay

  • Declare a world; the generator writes transition(state, action)
  • Gate 1 syntactic: parses and defines the required function
  • Gate 2 behavioral: sandboxed smoke-run, respects declared invariants
  • Gate 3 semantic (optional): a second model critiques against the rules
  • Failures loop back into the next prompt; accepted code is plain, editable source

A code world model, end to end

Perceive -> world -> emit/act, generated from a real spec so labels cannot drift
  • Perceptors turn any-modality inputs into symbolic state
  • Verified transitions step inside a composite of nested worlds
  • Objectives (dial-retuned) score; emit/act boundary returns reports and tool-calls

A composition algebra for worlds

  • Worlds are values; compose() is closed -- output is itself a world
  • Children run unmodified; bridges couple, aggregators roll up, bindings inject
  • Closure, identity, and associativity hold up to namespace renaming
  • Verification is compositional: verified pieces => verified whole
  • Turns one n-rule synthesis into n small, independently verified problems

Verified reward induction

  • Objective made symmetric to the transition: a verified reward as code
  • CodeObjective's reward() runs in the same pure-Python sandbox
  • induce_reward accepts only when it exact-matches held-out rewards
  • Closes the loop: verified perception + dynamics + reward + planning
  • The primitive ARC-AGI-3-style goal discovery demands

Experimental setup

Worlds with ground truth

  • Three deterministic oracle worlds: sprint, orchard, triage
  • Oracles are hand-written FunctionTransitions
  • Synthesized and learned engines compared against them exactly
  • Coding world: 20 buggy functions + hidden tests, test-execution dynamics
  • Composition/nesting/changing-rules probed by E30--E32

Models, hardware, baselines, statistics

  • All local via Ollama on an Apple-silicon laptop -- no cloud, no fine-tuning
  • qwen2.5 7b/3b/1.5b, plus llama3.1:8b and gemma2:9b for cross-family
  • Baseline: LLMTransition -- same backbone predicting each next state
  • Rate metrics get 95% Wilson CIs; judge test is exact paired McNemar
  • Seeds fixed; every number regenerates from one pipeline
Results: fidelity, latency, and trust

Performance: fidelity and latency (E04, E11)

  • 7B-synthesized dynamics exact on 24/24 20-step rollouts
  • Answer 10x out-of-distribution probes at 100%
  • No compounding error -- one verified transition reused at every step
  • 14,997 steps/s vs 0.32 for the LLM proxy (~47,000x)
  • Hand-written oracle is ~16x faster than synthesized code

Verifier ablation: correctness gained per gate (E3)

Blind 0.46 -> behavioral 0.65 -> +critic 0.70 (a +0.24 absolute gain)
  • Syntax-only adds nothing -- this generator's failures parse fine
  • No 3B generation is bit-exact under any regime
  • Verification buys correctness margin but cannot replace generator capability

Closing the false-accept gap (E62)

  • Single-state smoke-run has a blind spot: later-reached states
  • On 3 faults clean initially, single-state gate false-accepts 100%
  • Re-running over branch-covering states drops false-accepts to 0%
  • Cross-family (E16): Qwen/Llama/Gemma all reach verified acceptance
  • Gemma-2-9B is rule-perfect on every attempt; seconds per world

Agents-as-a-judge, controlled (E13+E17)

Judge 85% vs random 78%, first 78%, oracle ceiling 91% over 120 rounds
  • Judge recovers ~half the random-to-oracle gap (McNemar p marginal)
  • Order-reversal: raw choices match ~57% but accuracy is order-stable
  • Costs ~4x inference; judges should gate, not replace, verification

Anatomy of a composite world (E31)

Two countries, each with two cities, compose into a region -- real state values
  • Bridge carries trade under a conservation invariant
  • Route lets an agent cross and pay a toll; aggregators summarize upward
  • Binding injects a parent parameter into a child

Composition defeats the complexity cliff (E30)

16 rules: monolithic 0.31 vs four 4-rule children + bridges 0.92
  • Monolithic 7B synthesis collapses past ~8 coupled rules
  • Decomposition returns accuracy to near-R=4 levels
  • Acceptance identical in both conditions -- the decomposition is the difference

What an agent sees (E31)

Scoped observation: own node full, ancestors as aggregates, neighbors as summaries
  • Its own city fully observable; ancestor levels expose only _agg summaries
  • Route-adjacent city is a summary card; siblings not observable
  • Planners reason locally against exact dynamics without the macrostate

When the rules change, agents move (E33)

Austerity hits at step 9; agent pays the toll, relocates, gains +20 mobility
  • Shared economy enters austerity via PhasedTransition
  • Greedy agent prices the change and crosses to the growing economy
  • Both phased and monolithic synthesized code track the switch exactly

The sprint world: budget allocation (E34)

20 repair worlds, budget 80; fixed protocol solves 15/20
  • No policy beats the fixed ceiling -- the unsolved tail is capability-bound
  • Round-robin reaches the plateau in ~a third fewer attempts
  • Greedy closest-to-passing tarpits on a single task

A brain as a world-of-worlds (E58)

Conscious + unconscious + environment, coupled by retrieve/consolidate/act bridges

  • Perceive -> retrieve -> think -> act -> observe -> consolidate loop
  • The whole brain serializes to a spec and round-trips losslessly

Tree-of-thoughts ReAct with real memory (E58)

Persistent memory reaches optimal L=4; memoryless stays ~22, random flails
  • Persistent unconscious drives steps-to-goal to optimal in a couple episodes
  • Steps fall with planning depth, plateau once depth reaches the horizon
  • Bounded simulated lookahead helps up to the goal horizon, no further

The brain around a frozen model (E59)

Modeled backbone: bare 0.33 -> richest brain 1.00 (+0.68), no change to the LLM
  • Retrieval makes rare-fact questions answerable; best-of-N+verify wins hard ones
  • Ordering is a consequence of the modeling assumptions, not an empirical finding
  • With the model frozen, the world model around it is what you optimize

The perceive->world->emit->act boundary (E60)

Content-addressable recall routes 78% vs Jaccard 70% vs exact-key 0%; resolves 78% end to end
  • JSON/Regex perceptors + gates; MemoryStore; tool and code emitters
  • Contract gates catch every injected malformed input and out-of-contract output
  • All components serialize to a spec and round-trip with no LLM in the loop

Sprint allocation across a capability ladder (E35)

Fixed ceiling rises 15 -> 19/20 with capability; greedy tarpit defangs with scale
  • qwen2.5:7b -> deepseek-r1:14b -> gpt-oss:20b -> qwen3-coder:30b
  • Greedy tarpit is capability-gated; gpt-oss:20b clears all 20
  • Allocation policy is dominated by base model capability

Composition yields better representations (E36)

Every monolithic learner collapses on novel combos; the factored composite stays flat
  • Best monolith (gradient boosting) decays from 0.48 to 0.20 by K=5
  • Collapse holds across kernels, trees, GPs, and a Koopman operator
  • Factoring the representation -- not scaling the learner -- buys generalization

Adapting to sudden, unannounced rule changes (E41)

Symbolic monitor re-pins the rule in 1 step; sliding-window lags, frozen never recovers
  • A reservoir's hidden regime jumps at unannounced times
  • Explicit, identifiable program => a few observations re-pin it exactly
  • Holds while the changed rule stays in the identifiable family

Agents traversing connected worlds with changing rules (E42)

Per-world monitor 0.94 and returns nearly free; shared learner 0.85 pays interference
  • Two non-stationary worlds joined by a toll route; agent hops between
  • Per-world programs carry a world's rule across an absence intact
  • A single shared belief cannot hold two worlds at once

Next-token world models: exact length generalization (E45)

Induced program exact OOD on 3/3 tasks (1.00); same 30B predicting directly falls to 0.38
  • Parity, dyck, copy: memory a fixed window cannot hold
  • Synthesize-a-program mode is length-invariant by construction
  • n-gram and fixed-window MLP both decay with length

A database for many worlds (E46)

Exact version space over 1e18 worlds in 191 ms; #P boundary surfaced, not hidden
  • Likelihood factorizes over observables -> one small factor per mechanism
  • Pluggable semiring answers Boolean, counting, and probability queries
  • Couple w parameters non-separably and the factor grows as d^w

Path integrals over learning trajectories (E49)

Least-action curriculum reuses mastered sub-worlds -- a 6.0x speedup over from-scratch
  • Trajectory weighted by e^(-beta S); reuse the E46 semiring engine
  • Combinatorially many trajectories summed by DP without enumeration
  • Different starting roles (SWE, director, CEO) get different optimal paths

Abstract interpretation: sound, tight bounds (E54)

Affine bound width 21.5 vs interval 108.6; Monte Carlo tight but unsound
  • Run the world on abstract states -- propagate whole sets forward
  • Affine domain tracks a/(1-a) cancellation; intervals explode
  • Gives the world model a worst-case rollout it can certify

Information geometry: identify the world (E55)

Expected info gain identifies the true world in 6.5 probes vs random 9.4 (of 10 worlds)
  • Choose the probe whose outcome carries the most information
  • Adaptivity: stop re-confirming, separate worlds still in contention
  • Exact enumerable candidates make identification principled experiment design
Discussion and limitations

Interpretation: a structural trade

  • Learned models buy pixel breadth, pay in compounding error and opacity
  • Verified code buys exactness, speed, OOD transfer, auditability
  • Within the symbolic boundary the results are one-sided
  • The workflow: declare, verify with invariants, probe intent, tighten, recompile
  • The artifact stays a diffable program at every stage

What these worlds enable: world-time compute

  • Verified worlds label trajectories exactly, for free
  • Fine-tuning on many worlds of a domain lifts generalization to held-out worlds
  • This is world-time compute -- the subject of the companion paper
  • Exact, composable worlds are what make it possible
  • But a world computer is useful whether or not one ever fine-tunes

Value alignment as open specification

  • Objectives are declared artifacts -- the welfare--fairness frontier is operational
  • A weighted sum is one ethical theory (act consequentialism)
  • Aggregators, side-constraint vetoes, and a moral parliament are distinct objects
  • Vetoes cannot be simulated by weights (E25)
  • Open spec manages the specification trap procedurally, does not resolve it

Limitations

  • Scope: symbolic-state worlds only; pixel-native remains learned territory
  • Complexity: monolithic 7B synthesis unreliable past ~8 interacting rules
  • Scale: 20 repair tasks, 3 handwritten worlds; judges Qwen-only
  • Judge margin is marginal (~4x cost); ensemble self-check is a screen, not a proof
  • serve is a trusted-spec local endpoint -- no auth, sandbox is best-effort

Frozen perceptual features don't control cartpole (E67)

  • The perceptual species' best shot on cartpole still scores 0%
  • Five ways to bolt a controller onto frozen V-JEPA-2/DINOv2 all score 0
  • Yet a probe recovers the state: R2(sin theta) = 0.95
  • Strong frozen features are necessary but not sufficient for control
  • End-to-end task training, not better perception, closes the gap
Conclusion
Make the model a program, verify the program, and keep the values dialable -- an orthogonal path to scaling parameters until hallucination is suppressed.
