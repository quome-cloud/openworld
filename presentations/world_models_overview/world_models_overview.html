The basics

Two kinds of world programs

Picture-guessers

  • Big neural nets that dream up pixels
  • Dreamer, Genie, Sora
  • Need tons of data; mistakes pile up
  • You can't see inside

Code worlds

  • The rule for what's next is plain code
  • An AI writes it; we check it's right
  • Exact and easy to inspect
  • They help each other, not rivals

The toolkit

One agent writes worlds that are checked and ready to share

Measured on cost, speed, and quality; code we check is right ran about 47,000x faster than an LLM
  • Built 100 worlds, about 2.8 minutes each
  • 14,997 steps a second, versus 0.32 when asking an LLM each step
  • 100% pass the structure check; 99% behave correctly when run
The toolkit

A code world, start to finish

See -> world -> report/act, all built from a real spec so the labels cannot drift
  • Sensors turn any kind of input into the world as plain facts
  • Checked rules step forward inside big worlds made of small checked pieces
  • Goals (with turnable dials) give a score; the world sends back reports and tool-calls
The toolkit

The plan, write, check relay

1You describe a world
→
2A writer produces the rule: next = f(now, action)
→
3Three checks: it parses, runs safely, and a critic reads it
→
4Accept plain editable code; failures feed back into the next try
→
5Joining keeps the checks: checked pieces make a checked whole

Skill carries to new worlds as you add worlds (E84)

Held-out List Functions score climbs 29% -> 49% as the number of checked worlds grows
  • One learner trained on 128 separate worlds does well on 60 it never saw
  • A version with scrambled labels stays flat -- exact labels are what matter
  • A real growth curve, on data we did not make up
Three results

Across real subjects (E80)

Grows on rule-based subjects (List Functions, CLRS, ARC); flat on Bongard pictures
  • List Functions 35% -> 69%; ARC, practicing each world, +8 pts
  • Biggest help for short simple rules, smallest for long ones
  • Bongard picture task stays at guessing level -- the seeing barrier
Result III — Solving ARC-AGI-3
Three results

An agent that writes its own world program without reading the game's code

See -> explore -> write code we check is right -> figure out the goal -> plan -> replay to verify
  • Never reads the game's code; learns the rules by playing
  • Every win becomes a shareable OpenWorld world
  • Copying the world is exact -- the hard part is figuring out the goal
Three results

The hard part: the win is a recipe of steps, not one screen

A win is an ordered recipe A->B->C, which no single-screen score can judge
  • Four careful ways to guess the goal solve 0 -- even with a perfect world copy
  • Scoring one screen or blind searching cannot rank an ordered recipe
  • Only an agent that reasons out the win gets past the walls
Three results

The first perfect result without reading the code (bigger model)

Claude Fable clears all 25 games; a stronger model breaks more walls for the same budget
  • Claude Fable 5: 25 of 25 games, 183 of 183 levels -- first perfect run
  • Opus 4.8: 16/25; GPT-5.5: 12/25; best earlier baseline1: 15/25
  • Gets smarter and cheaper: Fable used about 3.5x fewer tokens
Three results

The atlas: 25 games, 25 shareable worlds

Every win found without reading the code turns into a runnable OpenWorld world
  • hidden-status-bar sensor -> code rule from what was learned -> code goal
  • Runs at openworld serve /view; draws itself as an atlas card
  • 25 clearly different worlds, all discovered by playing

Where it is useful
Physics, economies, brains, signals
Where it is useful

Einstein's relativity as a checked world program (E47)

GPS fix of +38.5 us/day and the Hafele-Keating clock shifts, from stated physics
  • Time slowing is exact; a slow-speed guessed fit goes wild as speed nears light
  • Stated, checked law where guessed video models miss the physics
Where it is useful

An economy that emerges from joined rules (E44)

6 people; inequality 0.22 -> 0.001 with sharing; the books balance exactly every tick
  • Big-picture numbers are just sums built from small checked parts
  • Each effect isolated by flipping one checked rule on or off
Where it is useful

A brain as a world made of worlds (E58)

Awake mind + hidden mind + surroundings, linked by recall, store, and act bridges
  • Short-term memory + searchable long-term memory + the world outside
  • The whole brain saves to a spec and rebuilds with nothing lost
Where it is useful

See it, then forecast an outbreak (E40)

The seen world is exact both on and off familiar data; the neural-net fit falls apart at 10x scale
  • Text plus a photo read into healthy/sick/recovered counts, then rolled 14 days ahead
  • The rules add zero error -- the forecast is as good as the reading was
Where it is useful

A same-day stock trading world (E50)

Real prices, tested forward in time, after costs -- about rigor and audit, not easy money
  • It finds a known fake +40 bps edge, so a weak real result is trustworthy
  • Honest and fragile to costs: score 0.59 < the market's 0.77
Where it is useful

Portable, auditable world specs and cards (E57)

One self-contained picture drawn from a world's spec; rebuilds with nothing lost
  • Facts, rules, dynamics, seeing/reporting, goals -- all captured
  • Exports to Mermaid and React Flow; a tradeable unit for worlds
