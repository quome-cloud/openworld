Why we did this

The problem: real practice is hard to get

Needs examples

AI gets good only after many real examples with the right answers

Rare and costly

In most areas those examples are rare and expensive

A cheap fix

We found a cheap way to make them

The trick

When a world's rules can be written as code, we simulate the practice

Why we did this

Two kinds of world model

Picture kind

  • Big learned nets guess pixels or video (Dreamer, Genie, Sora)
  • Need tons of data
  • Small mistakes pile up
  • You can't read what's inside

Code kind (CWMs)

  • Rules are a plain program you can read, over simple facts
  • An AI writes the program
  • We check it is right
  • Then it runs exactly and needs no training data
Picture-based systems try to learn things nobody can put into words. OpenWorld writes down the rules that CAN be put into words -- and then checks them.

The core idea

Checked code stops mistakes from piling up

A: code stays exact all the way; guessing drifts off by about step 2.3. B: the payoff of making practice worlds.
  • Code (blue) matches the true answer exactly across all 20 steps
  • Guessing each step (orange) first drifts around step 2.3 and never finishes exactly
  • B: extra training on 60 checked worlds raises the score on worlds it never trained on
  • The smallest models gain the most: +29 pts at 0.5B, +3 at 32B
The core idea

Head-to-head: how true the rules stay (E1/E4/E11)

Right once means right at every depth; the step-by-step guesser never finishes exactly
  • Code runs at about 15,000 steps a second; the guesser at 0.32 steps a second
  • On surprise cases: code is 100% exact with zero practice steps
The claim you can trust is the exact-match gap, not the exact step where guessing drifts -- how fast mistakes pile up depends on each world.

The core idea

Inside one diagnosis world (E74)

A hidden disease gives off symptoms; the agent works backwards to name the disease.
  • Each specialty is a tiny simulator with a hidden cause and a known answer
  • The agent uses its own checked world as the source of truth
  • Practice on many specialties, then test on ones it never saw
  • This forces it to learn the real skill, not just memorize
The core idea

It really generalizes -- and small models gain most

A boost at every model size on worlds it never saw; the smaller the model, the bigger the boost
  • 0.5B: 37% -> 66%; 32B: 78% -> 81%, closing on the 86% you get when told the rules
  • Worlds that share no common skill (E71/E73) give no carry-over

Scaling and real data

More worlds keeps helping (E76)

Score on unseen worlds versus how many worlds you practice on; still rising at 512.
  • Too few worlds hurts (below 48%) -- it overfits a narrow slice
  • After that the score keeps climbing, no flattening out
  • Still rising at 512 worlds: 59% (+10 pts), heading toward the 69% you get when told the rules
  • The number of worlds is a real knob you can turn
Scaling and real data

Correct answers are the real lever (E78b)

Spoil a fraction p of the training answers; the score drops steadily.
  • 256 worlds, held fixed; only the training answers get corrupted at rate p
  • All correct (p=0): 35%; all wrong (p=1): 19%, at or below the floor
  • Correct answers -- not just having lots of tasks -- is what drives it
  • This is what sets our approach apart from just shuffling the tasks

Real Data: Practice Worlds on ARC-AGI
Scaling and real data

How we train on one checked world (E80)

An ARC puzzle becomes many correctly-answered rows, then extra training on a frozen 4-bit Qwen2.5-7B.
  • Each ARC puzzle is a world with a hidden rule shown by examples
  • Rotating and recoloring keeps the rule but makes endless correct rows
  • Two strict tests: a new-puzzle ladder, and practicing on each world at answer time
  • The spoiled-answer version is the control that tells the difference
Scaling and real data

Practice worlds on real ARC-AGI (E80)

Practicing on each world at answer time raises exact matches; carrying skill to new puzzles stays weak.
  • Cold start 2% -> light practice 8% -> heavy practice 10% (+8 pts) with more compute
  • Spoiled examples fall to 2%, back down at the cold-start floor
  • The boost needs correct answers -- the model learns each real rule
  • Carry-over to new puzzles is weak (3% -> 5%): every ARC rule is brand new
Scaling and real data

Across real tasks and kinds of data (E80)

It scales on plain-fact tasks and stalls on vision; Bongard picture puzzles stay at a coin flip.
  • List Functions: 35% -> 66% -> 69% (heavy); spoiled 3%
  • CLRS algorithms: 9% -> 13% -> 17%; spoiled 4%
  • Biggest gains on short rules, smallest on long step-by-step algorithms
  • Bongard picture puzzles: flat at a coin flip (46/48/52%) -- the vision wall
A Skill That Carries to New Worlds
Scaling and real data

One skill across many worlds, List Functions (E84)

One shared add-on trained on 128 separate worlds works on 60 worlds it never saw.
  • 40% (range [35,45]) on worlds it never saw
  • The spoiled-answer control gets only 6% -- a +34-pt gap that needs correct answers
  • The gap is real rule-figuring carrying over, not just formatting
  • The new idea, shown on data we did not make ourselves
Scaling and real data

The carry-over grows with more worlds (E84)

Score on unseen worlds rises 29% -> 49%, then levels off near 64 worlds.
  • Rises from 29% at 16 worlds to a 49% peak by 64 worlds
  • The spoiled-answer control stays flat
  • A rise-then-level-off curve on real data
  • More checked worlds means more carry-over

Part 4
Where it breaks
Where it stops working, and why
Where it breaks

Different wording doesn't carry over (E81)

Practicing in other languages doesn't beat a cold start on either model.
  • One problem in 5 languages; practice on some, test on a held-out language
  • Strong coder flat (28% -> 30%); weak one drops (29% -> 27%)
  • Only a weak correct-answers ordering survives (correct beats spoiled)
  • A new language acts like its own separate skill -- an honest limit
Where it breaks

The mix-and-retry loop (E82)

Use the checked world to make examples, plus a check-and-try-again safety net on HumanEval-X.
  • Pre-baking examples helps a strong model (80% -> 85%) but hurts a weak one (57% -> 50%)
  • The mix-and-retry loop lifts both: Qwen 90% (+10), Llama 65% (+8)
  • The weak model leans on the checker more (2.65 vs 1.45 tries per problem)
  • Just using the checker as a yes/no judge is enough -- the safe default
Against Trained and Picture-Based Rules
Where it breaks

Trained baselines vs a zero-practice program (E12)

A trained net and a nearest-example lookup on K steps; neither works on surprise cases.
  • With 10,000 steps the trained net hits only 50% on familiar cases, 0% on 10x-out cases
  • The written program: 100% on both, from zero practice steps
  • Nearest-example lookup replays known runs (8/8) but scores 30% on new branches
  • Surprise cases and new branches, not replaying known runs, show real quality
Where it breaks

Bigger models don't figure rules out better from examples (E38)

Figuring rules from examples alone stays flat across three model families.
  • From examples alone (no rule text): 0.43 (7B), 0.33 (30B coder), 0.43 (gpt-oss)
  • None comes near the 1.00 you get when handed the rule text
  • Every model does the same on familiar and 10x-out cases
  • The examples just don't pin the rule down -- it's not about model size
Where it breaks

Trying moves beats what size can't (E43)

Picking smart moves to test beats just watching.
  • Smart poking finds the exact rule in 11.2 steps vs 14.7 by just watching
  • Just watching never nails 2 of 5 tricky rules within budget
  • Smart poking (knowing nothing) matches a see-everything ideal (11.2 vs 11.6)
  • When examples aren't enough, choose which moves to test
Where it breaks

The complexity cliff (E20)

How often the build succeeds versus how many tangled rules you ask for.
  • R=4: 1.00; R=8: 0.53; R=12: 0.25; R=16: 0.15
  • One-shot 7B building gets shaky past about 8 tangled rules
  • The current limit at this model size
  • Points to building it piece by piece -- one expert per rule

Stepping back

Isn't this just extra training?

Yes, but

Under the hood it's ordinary extra training; the point is where the data comes from

Compute to data

A correct world model turns compute into endless correct training data

Earns the name

Two things: it needs correct answers, and the skill carries to new worlds

Honest limit

Where neither holds, it's just costly extra training -- and we say so

Stepping back

We measured the unit of practice (E85)

The unit = one step

You train on one step (state, move, next state), not a final answer

In-range +12 pts

At the trained length: full-path 70% vs final-answer 58%

Longer: catches up

Run longer than trained, final-answer catches up (75% vs 68%)

Pile-up returns

Stringing step guesses brings back the pile-up on the learned side

Stepping back

When is it worth it? Short reasoning

Wins when

A task is a few steps over facts you can spell out

Sweet spot

Fades when

Reasoning gets longer (CLRS traces, more than 8 tangled rules)

Leaves to learned

Long, picture-heavy problems go to the learned models
