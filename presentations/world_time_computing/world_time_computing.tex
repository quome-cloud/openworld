{\color{owochre}\bfseries\footnotesize PART 1}\\[5pt]
{\color{owdeep}\bfseries\Large Why we did this}\\[10pt]{\color{owmuted}\large Making practice out of thin air}\vfill\end{frame}
\begin{frame}{The problem: real practice is hard to get}
\begin{itemize}
  \item AI gets good only after seeing many real examples with the right answers
  \item In most areas those examples are rare and expensive
  \item We found a cheap way to make them
  \item The trick: when a world's rules can be written as code, we can simulate the practice
\end{itemize}
\end{frame}
\begin{frame}{Two kinds of world model}
\begin{itemize}
  \item The picture kind: big learned networks that guess pixels or video (Dreamer, Genie, Sora)
  \item They need tons of data, small mistakes pile up, and you can't read what's inside
  \item The code kind (CWMs): the rules are a plain program you can read, over simple facts
  \item An AI writes the program, we check it is right, then it runs exactly and needs no training data
\end{itemize}
\end{frame}
\begin{frame}[plain]\centering\vfill
{\color{owdeep}\bfseries\LARGE Picture-based systems try to learn things nobody can put into words. OpenWorld writes down the rules that CAN be put into words -- and then checks them.\par}\vfill\end{frame}
\begin{frame}{Two things have to be true for this to work}
\begin{itemize}
  \item The practice worlds must never lie: their rules are checked code, not a guess
  \item We only keep a world if it reads cleanly, runs safely, keeps its promises, and passes a reviewer
  \item You need many worlds -- so each one has to be cheap to build
  \item OpenWorld lets you build a checked world in minutes
\end{itemize}
\end{frame}
\begin{frame}{What we contribute}
\begin{itemize}
  \item Turning computer time into practice on real tasks, plus a skill that carries to new worlds
  \item Why it has to be checked code: it runs exactly, so small mistakes never pile up
  \item The OpenWorld toolkit, fully repeatable (79 experiments, fixed random seeds)
\end{itemize}
\end{frame}
\begin{frame}[plain]\centering\vfill
{\color{owochre}\rule{40pt}{2.5pt}}\par\vskip10pt
{\color{owdeep}\bfseries\Large Setting up the problem\par}\vfill\end{frame}
\begin{frame}{Worlds made of plain facts}
\begin{itemize}
  \item The state is simple data; the moves are a short list; the rules can be spelled out
  \item Covers things like triage, bargaining, portfolios, sprints, fixing programs
  \item Not picture-heavy worlds -- that is where this stops
  \item The enemy is a wrong model: made-up moves, code that is quietly wrong
\end{itemize}
\end{frame}
\begin{frame}{What a world model is, plainly}
\begin{itemize}
  \item A world = its states, its moves, a start, a rule for what happens next, goals, and promises it must keep
  \item The person spells out everything except the next-step rule (facts, moves, rules, dials)
  \item The tool writes the next-step rule by building and checking it, not by learning it
  \item It proves the rule keeps its promises, not that it matches the true world perfectly
\end{itemize}
\end{frame}

\begin{frame}[plain]\centering\vfill
{\color{owochre}\rule{40pt}{2.5pt}}\par\vskip10pt
{\color{owdeep}\bfseries\Large How we ran the tests\par}\vfill\end{frame}
\begin{frame}{The worlds and the models}
\begin{itemize}
  \item Three known-answer worlds: sprint, orchard, triage (we wrote the true rules)
  \item A coding world: 20 buggy functions to fix, where running the tests decides what happens
  \item Local models: qwen2.5 1.5B/3B/7B, plus Llama-3.1-8B, Gemma2-9B
  \item Extra-training runs: one cloud A100, low-memory 4-bit method
\end{itemize}
\end{frame}
\begin{frame}{What we compared against, and how we scored}
\begin{itemize}
  \item Same model guessing each next step directly (stand-in for the learned way)
  \item A trained net and a nearest-example lookup on 100 / 1k / 10k random steps
  \item Did it match exactly, where did it first go wrong, how far off at the end; pass@k for fixing code
  \item 95\% confidence ranges; every number comes back out of one script
\end{itemize}
\end{frame}

\begin{frame}{More worlds keeps helping (E76)}
\centering

\begin{frame}{Correct answers are the real lever (E78b)}
\centering
\includegraphics[width=\linewidth,height=0.48\textheight,keepaspectratio]{noise_ablation.png}\vskip3pt{\color{owmuted}\footnotesize Spoil a fraction p of the training answers; the score drops steadily.\par}

  \item All correct (p=0): 35\%; all wrong (p=1): 19\%, at or below the floor
  \item Correct answers -- not just having lots of tasks -- is what drives it
  \item This is what sets our approach apart from just shuffling the tasks
\end{itemize}}

\begin{frame}{A family of coding worlds (E77)}
\begin{itemize}
  \item 164 write-this-function worlds, each checked by its own tests
  \item Same kind of task: 7B pass@5 rises 84\% -> 95\% at every size
  \item On outside tasks (HumanEval), it hurts the single greedy try (78\% -> 70\%)
  \item But it helps pass@5 (87\% -> 91\%) from just 164 worlds
\end{itemize}

\begin{frame}{How we train on one checked world (E80)}
\centering
\includegraphics[width=\linewidth,height=0.48\textheight,keepaspectratio]{arc_method.png}\vskip3pt{\color{owmuted}\footnotesize An ARC puzzle becomes many correctly-answered rows, then extra training on a frozen 4-bit Qwen2.5-7B.\par}

  \item Each ARC puzzle is a world with a hidden rule shown by examples
  \item Rotating and recoloring keeps the rule but makes endless correct rows
  \item Two strict tests: a new-puzzle ladder, and practicing on each world at answer time
  \item The spoiled-answer version is the control that tells the difference
\end{itemize}}
\end{frame}
\begin{frame}{Practice worlds on real ARC-AGI (E80)}
\centering
\includegraphics[width=\linewidth,height=0.48\textheight,keepaspectratio]{arc.png}\vskip3pt{\color{owmuted}\footnotesize Practicing on each world at answer time raises exact matches; carrying skill to new puzzles stays weak.\par}
\flushleft \vskip3pt{\footnotesize \begin{itemize}
  \item Cold start 2\% -> light practice 8\% -> heavy practice 10\% (+8 pts) with more compute
  \item Spoiled examples fall to 2\%, back down at the cold-start floor
  \item The boost needs correct answers -- the model learns each real rule
  \item Carry-over to new puzzles is weak (3\% -> 5\%): every ARC rule is brand new
\end{itemize}}

\begin{frame}{A simple pattern we noticed (E80)}
\begin{itemize}
  \item The learning curve tapers off: score(c) = base + (top - base)(1 - e\textasciicircum{}{-c/kappa})
  \item The top it reaches depends on whether the rule can be pinned down and reached, and how much room is left
  \item A quick small dose predicts the eventual top (Spearman 0.73-0.98)
  \item It also holds for a non-AI program-builder (so it is not about one learner)
  \item Worlds needed to reach 90\%: List Functions 3, ARC 9, CLRS 15 -- harder needs more
\end{itemize}
\end{frame}

\begin{frame}{Isn't this just extra training?}
\begin{itemize}
  \item Under the hood, yes -- it's ordinary extra training; the point is where the data comes from
  \item A correct world model turns compute into endless correct training data
  \item Two things earn the name: it needs correct answers, and the skill carries to new worlds
  \item Where neither holds, it's fairly just costly extra training -- and we say so
\end{itemize}
\end{frame}
\begin{frame}{We measured the unit of practice (E85)}
\begin{itemize}
  \item The unit you train on is one step (state, move, next state), not a final answer
  \item At the length it trained on: full-path 70\% vs final-answer 58\% (+12 pts)
  \item Run longer than it trained, the final-answer way catches up (75\% vs 68\%)
  \item Stringing together step guesses brings back the pile-up on the learned side
\end{itemize}
\end{frame}
\begin{frame}{When is it worth it? Short reasoning}
\begin{itemize}
  \item It wins when a task is a few steps over facts you can spell out
  \item Short rule-figuring, small grid changes, planning a few moves ahead
  \item It fades as the reasoning gets longer (CLRS traces, more than 8 tangled rules)
  \item It leaves long, picture-heavy problems to the learned models
\end{itemize}
\end{frame}
